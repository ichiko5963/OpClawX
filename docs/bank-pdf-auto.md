# 銀行明細PDF自動処理 - 使い方

## 📄 仕組み

1. **TelegramにPDF送信**
2. **私（Claude）がPDFを読む**
3. **入金のみ抽出**（振込は除外）
4. **収入シートの一番上の空白行に追加**

---

## 💡 入金/支出の判定

### ✅ 入金（追加される）
- 外国非課税送金、ガイコクヒシムケソウキン
- Stripe、ストライプ
- Note、ノート
- Port、ポート
- メルカリ、メルペイ
- 九州大学、キュウシュウダイガク
- 利息、オリソク
- キャリアデザインセンター、みん就、Lily等

### ❌ 支出（除外される）
- 振込、フリコミシキン、フリコミテスウリョウ
- ATM、ATMシハライ、ATMテスウリョウ
- クレカ、カード、ミツイスミトモカード
- RS ペイペイ、楽天カード、ライフカード

---

## 🚀 使い方

### Telegramから自動処理
1. 銀行明細PDFをTelegramで送る
2. 私が自動で：
   - PDFを読む
   - 入金を抽出
   - 収入シートに追加
3. 「✅ 25件の入金を追加！総額4,439,221円」と報告

### 手動テスト
```bash
cd ~/Documents/OpenClaw-Workspace
source venv/bin/activate

# JSON形式で入金データを渡す
python3 tools/bank_pdf_auto.py bank.pdf '[
  {"date":"2026/01/06","source":"外国非課税送金","amount":"28522"},
  {"date":"2026/01/14","source":"メルカリ","amount":"123600"}
]'
```

## 🔧 技術詳細

### スクリプト
- `tools/bank_pdf_auto.py` - メイン処理

### 判定ロジック
1. 取引摘要をチェック
2. 支出キーワードがあれば除外
3. 入金キーワードがあれば追加
4. どちらでもなければ除外（安全側）

### 追加先
- スプレッドシート: `1FH_CZkEkn621MNvFioUHgT3_4UU_TL1POu-Bhpz7KCc`
- シート: `2026年収入`
- 挿入位置: 一番上の空白行（上から詰める）

---

## ⚠️ 注意点

- **重複チェックなし**: 同じPDFを2回送ると2回追加される
- **日付順ではない**: 送った順に上から追加される
- **手動確認推奨**: 追加後にスプレッドシート確認
