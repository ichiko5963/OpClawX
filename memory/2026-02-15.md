## 🎯 達成：完全自動領収書処理システム

### システム概要
- **LINE公式アカウント**: PLai領収書管理（ID: 2009132576）
- **OCR**: OpenAI GPT-4o（Vision）
- **処理**: 画像送信 → 自動分析 → Drive保存 → Sheet追記 → 全情報表示 → 修正機能
- **webhook**: ngrok ポート5001

### 🆕 実装した機能

#### 1. OpenAI GPT-4V統合
- Anthropic → OpenAI に変更
- API Key: `OPENAI_API_KEY` 環境変数（~/.zshrc）
- モデル: `gpt-4o`
- コスト: 画像1枚 約$0.01

#### 2. 新列構造（A-I、9列）
1. 日付
2. 宛名
3. **カテゴリー**（GPT-4Vが自動判定）
4. 発行者の名前と住所
5. メモ
6. 金額
7. 通貨
8. URL
9. 振り込み期日

#### 3. カテゴリー自動判定
**最優先ルール（重要！）:**
- **発行者に個人名（山田太郎、ハタ コウタ等）がある**
- → **必ず「従業員請求書」カテゴリー**
- → **内容に関わらず最優先適用**

その他カテゴリー:
- 交通費、消耗品費、通信費、広告宣伝費
- 外注費、会議費、接待交際費
- 備品購入、書籍・資料費、その他経費

#### 4. 🎯 修正機能（超重要！）
**フロー:**
1. 画像送信 → 全9列の情報を返信表示
2. 「間違いがあればメッセージで教えてください」と促す
3. ユーザーがテキスト送信（例：「カテゴリーは交通費」）
4. GPT-4が修正内容解析
5. スプレッドシート自動更新
6. 「✅ 修正しました」返信

**セッション管理:**
- 最後の処理情報を `/tmp/line_sessions/{user_id}.json` に保存
- テキスト受信時に参照して該当行を更新

**修正例:**
- 「カテゴリーは交通費」
- 「金額は5000円」
- 「期日は2026/03/01」

#### 5. 返信フォーマット
```

例: 「カテゴリーは交通費」「金額は5000円」

Drive: https://...
```
