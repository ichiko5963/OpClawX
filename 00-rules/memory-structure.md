# メモリ構造ルール

## 階層構造

```
ブートストラップ層（毎セッション読み込み）
├── MEMORY.md — CEOハブ：会社・事業・役割マップ + ポインタ
├── SOUL.md / USER.md / IDENTITY.md — 人格・ユーザー情報
├── HEARTBEAT.md / TOOLS.md — 運用設定

ルール層
├── 00-rules/ — 全ドメインの行動規範

個人知識層
├── 01-private/ — プライベート知識（信念・口調・人間関係・日記・学び）

事業層
├── 02-business/ — 会社→自社/他社→事業→プロジェクト

日次ログ層
├── memory/ — OpenClawデフォルト互換（AI作業ログ）
```

## 命名規則

- 日付入りファイル: `YYYY-MM-DD-[内容].md`
- 月次まとめ: `YYYY-MM.md`
- ルールファイル: `_RULE.md`（各フォルダの先頭に配置）
- プレフィックス `_` はメタファイル（ルール・インデックス・テンプレート）

## P0/P1/P2 優先度運用

各エントリに優先度タグを付与可能：
- `[P0]` 永遠：コアルール・ユーザーの好み。期限なし
- `[P1]` 90日：進行中プロジェクトの戦略。90日でアーカイブ検討
- `[P2]` 30日：デバッグメモ・一時イベント。30日でアーカイブ

## MEMORY.md の管理原則

- 軽量に保つ（50行以下を目標）
- ニュース・詳細データは c-suite/ceo/source-knowledge/ へ
- 事業・プロジェクト一覧は常に最新に保つ
- 重要な意思決定のみ記載
