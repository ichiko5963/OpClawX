# プロジェクトライフサイクル

## 定義

- **事業** = 継続的な活動（X運用、Peatixイベント運営など）
- **プロジェクト** = 金銭が発生する具体的な案件（事業の中から生まれる）

## 新規プロジェクト検出トリガー

以下の状況で新プロジェクトが検出された場合、自動でフォルダを生成する：

1. **MTG文字起こし**（`02-business/mtg-inbox/`）に新案件名が登場
2. **チャット**でユーザーが新プロジェクトを言及し、金銭発生が明確
3. **メール**（CFO経由）で新案件が発生

## 自動生成手順

1. 該当事業の `projects/` 配下にフォルダ作成
2. 以下のファイルを作成：
   - `overview.md` — プロジェクト名・目的・開始日・クライアント
   - `backlog.md` — タスク一覧
   - `people.md` — 関係者一覧（CHRO管轄）
   - `mtg/` — プロジェクト固有のMTG議事録
   - `deliverables/` — 成果物（役割ごとのサブフォルダ）
3. MEMORY.md のアクティブプロジェクト表に追加
4. CHRO の `project-assignments.md` を更新
5. `memory/YYYY-MM-DD.md` に「新プロジェクト作成」を記録

## MTG文字起こしの処理フロー

```
mtg-inbox/ に投入
    │
    ├── 既存事業のMTG → 該当事業の mtg/ に移動
    ├── 既存プロジェクトのMTG → 該当プロジェクトの mtg/ に移動
    ├── 新プロジェクトの話題 → 上記手順でプロジェクト生成
    ├── 財務の話題 → CFOのknowledge/に追記
    ├── 営業の話題 → CSOのknowledge/に追記
    └── アクションアイテム → backlog.mdに追加（担当C-Suite付き）
```

## アーカイブ条件

- 3ヶ月以上動きがないプロジェクト → `archive/old-projects/` に移動
- MEMORY.md から削除し、archive記録に追記
- 事業自体は基本的にアーカイブしない（事業の中のプロジェクトのみ）
