# スキル作成ルール

## スキルとは

スキル = AIが特定のタスクを実行するための**再現可能な手順書**。
以下の3要素で構成される：

1. **手順（How）** — 何をどの順序で実行するか
2. **参照ナレッジ（What）** — どの知識ベースを見るか
3. **出力先（Where）** — 成果物をどこに保存するか

## スキルの保存場所

```
skills/
├── _RULE.md              ← このファイル
├── [skill-name]/
│   ├── SKILL.md           ← スキル定義（必須）
│   └── [補助ファイル]      ← プロンプト、テンプレート等
```

## SKILL.md テンプレート

```markdown
# [スキル名]

## 概要
このスキルが何をするかを1-2文で説明。

## 担当C-Suite
- 主担当: [CMO / CTO / CSO / CFO / CHRO]
- 副担当: [あれば]

## 参照ナレッジ
| パス | 内容 |
|------|------|
| `02-business/c-suite/[role]/knowledge/[file]` | [何の知識か] |
| `02-business/[company]/[business]/[file]` | [事業固有の情報] |
| `01-private/[path]` | [個人知識（必要な場合のみ）] |

## 参照ルール
- `00-rules/[role].md` — 役割固有のルール
- `00-rules/deliverable-flow.md` — 成果物の蓄積フロー

## 実行手順
1. [ステップ1]
2. [ステップ2]
3. ...

## 出力先
- 成果物: `02-business/[company]/[business]/[deliverables-path]/`
- ナレッジ蓄積: `02-business/c-suite/[role]/knowledge/`

## 品質基準
- [この成果物が満たすべき条件]

## 改善ログ
| 日付 | 変更内容 | 理由 |
|------|----------|------|
```

## スキル作成のトリガー

以下の場合にスキルを作成する：
1. 同じタスクを**2回以上**手動で実行した
2. ユーザーから「これを自動化して」と指示された
3. C-Suiteの作業で**繰り返しパターン**を検出した
4. MTGから**定期タスク**が特定された

## スキル改善サイクル

1. **実行** — スキルに従ってタスクを実行
2. **記録** — 成果物の品質・所要時間・問題点を記録
3. **改善** — 改善ログに追記し、手順を更新
4. **ナレッジ化** — 汎用的な学びを c-suite/[role]/knowledge/ に蓄積

## スキルマップ（skill-map）

各C-Suiteが持つスキルの一覧を可視化する：

```markdown
## CMOスキルマップ
- X投稿作成 → skills/x-bazu-prompt/
- 記事作成 → [未作成]
- Peatixイベントページ作成 → [未作成]
- 画像生成 → [未作成]

## CTOスキルマップ  
- 自動化スクリプト作成 → [未作成]
- Next.js開発 → [未作成]
- デプロイ → [未作成]
```

スキルマップは `02-business/c-suite/[role]/_RULE.md` 内に記載する。
新スキル作成時、該当役割の _RULE.md のスキルマップも更新すること。
