# 8週間 OpenClawワークショップ ── 大まかなカリキュラム

8週間の流れを**週単位**でまとめた概要。審査面談・LP・特商法用のたたき台として使用。

---

## 全体コンセプト

**「AI従業員を増やして、少人数で事業を回す」**

- Step1 メモリ構築（Week1〜2）
- Step2 OpenClaw自動化・知識インプット（Week3〜4）
- Step3 マーケティング活用（Week5〜6）
- Step4 業務自動化〜AI従業員構築（Week7〜8）

---

## Week 1：OpenClawの環境構築（使えるようになろう）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | OpenClawを触れる状態にする |
| **狙い** | 環境構築完了／チャットで指示が通ることを体験する |
| **主な内容** | OpenClawのインストール・初期設定／APIキー・接続確認／「話しかけて動く」まで |
| **宿題例** | 環境構築のメモ作成／簡単なタスクを1つ依頼して結果を共有 |

---

## Week 2：メモリ構築をしよう！（ObsidianとCursor）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | 「第二の脳」をObsidianとCursorで作る |
| **狙い** | マークダウンで知識を整理する／AIに渡す“材料”を貯める土台を作る |
| **主な内容** | Obsidianの基本／マークダウンでのメモ・見出し・リンク／Cursorでの編集・検索／メモリの設計思想（何をどこに書くか） |
| **宿題例** | 自分の事業・強み・判断基準を1本の .md にまとめる |

---

## Week 3：メモリの環境構築！（OpenClaw編：Memory Systems作成）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | OpenClaw側で「メモリ」を読み込ませる |
| **狙い** | Obsidian等で作ったメモリをOpenClawに渡し、自分らしい返答・判断をさせる |
| **主な内容** | OpenClawのメモリ機能（memory.md 等）／参照させるファイル・フォルダの設計／Memory Systemsの作成と動作確認 |
| **宿題例** | 自分用のMemory Systemを1つ作り、簡単な質問で挙動を確認して報告 |

---

## Week 4：【基礎〜応用】知識インプットDAY（GitHubやVercelの基礎！開発をできるように！連携をできるように！）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | 開発・連携に必要な基礎知識を一気にインプット |
| **狙い** | GitHub・Vercelの基礎を理解し、「開発」「連携」をチャットで指示できる土台を作る |
| **主な内容** | GitHubの基礎（リポジトリ・Push・branch）／Vercelの基礎（デプロイ・URL公開）／OpenClawやCursorとの連携イメージ／「コードを書かずに作る」感覚 |
| **宿題例** | 簡単なLPやページを「作ってVercelに上げて」と依頼してURLを提出 |

---

## Week 5：情報の一元管理（GitHub Actionsでデータ自動収集）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | 情報を1か所に集め、自動で更新される仕組みを作る |
| **狙い** | バラバラのデータを一元化し、AIが参照しやすい形にする |
| **主な内容** | 情報の一元管理の考え方／GitHub Actionsでデータを自動収集する流れ／収集したデータの保存形式（.md・CSV等）／OpenClawが参照する“正解の本”の更新 |
| **宿題例** | 自分が集めたいデータを1種類決め、収集フローを設計して共有 |

---

## Week 6：コンテンツビジネスへのマーケティング活用フル講義（X, note, メルマガ, LP作成）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | コンテンツ・集客をOpenClawで効率化する |
| **狙い** | X・note・メルマガ・LPを「AIと一緒に作る」設計ができるようになる |
| **主な内容** | X運用の自動化（バズ分析→投稿案生成）／note・メルマガの下書き・構成案／LP作成（チャットで指示→Vercelで公開）／集客導線の設計（X→セミナー→サービス） |
| **宿題例** | 自分の事業向けに「1本のX投稿案」または「LP1枚」をOpenClawに作らせて提出 |

---

## Week 7：チャットで開発！自分の事業の業務自動化（領収書、日報、メール返信等）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | 自分の事業の“面倒な業務”をチャットで開発して自動化する |
| **狙い** | 領収書・日報・メール返信など、1つでも「OpenClawに任せる」形にする |
| **主な内容** | 業務自動化の設計（何を誰がやるか）／領収書整理・経費まわり／日報の自動作成／メール返信の下書き・分類／既存ツール（LINE・Slack等）との連携イメージ |
| **宿題例** | 自分で「自動化したい業務」を1つ選び、OpenClawへの指示文と結果（または設計メモ）を共有 |

---

## Week 8：役割別AI従業員の設計（営業・リサーチ・経理・マーケ）＋成果報告（OpenClawで作ったスライド構成で成果を共有）

| 項目 | 内容 |
| --- | --- |
| **テーマ** | 複数台・役割分担で「AI従業員軍団」を設計し、8週間の成果を共有する |
| **狙い** | 営業・リサーチ・経理・マーケなど役割別にAIを割り振る設計ができる／受講者同士で成果を見せ合う |
| **主な内容** | 役割別AI従業員の設計（営業・リサーチ・経理・マーケ）／どの業務をどのAIに任せるか／情報の一元管理と複数OpenClawの連携／成果報告：OpenClawで作ったスライド構成で成果を共有 |
| **宿題例** | 8週間でやったこと・作ったもの・今後やりたいことをスライド（または .md）にまとめて発表 |
