# 8週間 OpenClawワークショップ ── 申込LP作成プロンプト（面談後・決済用）

以下の指示に従い、審査面談を通過した方が閲覧する「申込・決済用LP」を1枚のHTMLページとして作成してください。
このLPは面談後に送られるリンクです。読者はすでに面談でワークショップの概要を聞いており、「申し込むかどうか」を最終判断する段階にいます。

---

## 目的

このLPの目的は「面談を受けた方が、自分に合ったプランを選び、決済ページへ進む」ことです。
面談で聞いた内容を改めて整理し、「やはり自分に必要だ」と確信を持って決済ボタンを押せる状態にする。
シンプルすぎず、でも冗長にもならず、「8週間で何が手に入るか」と「どのプランが自分に合うか」が明確にわかるページにしてください。

---

## ターゲット

- 審査面談を通過した方（＝すでに面談でワークショップの全体像・料金・審査制の趣旨を聞いている）
- 面談後24時間以内に決済するかどうかを判断する段階
- ユニコスクール生の方と、初めての方が両方いる

### セクション3：8週間のカリキュラム（詳細版）

**要素**
- セクション見出し：「8週間で何を、どの順番で作るか。」
- 4ステップ概要（大きなフロー表示）：
  - Step 1 メモリ構築（Week 1〜2）
  - Step 2 OpenClaw自動化・知識インプット（Week 3〜4）
  - Step 3 マーケティング活用（Week 5〜6）
  - Step 4 業務自動化〜AI従業員構築（Week 7〜8）

- 各Weekの詳細カード（アコーディオンまたは展開済みカード、全8枚）：

**Week 1：OpenClawの環境構築**
- テーマ：OpenClawを触れる状態にする
- 内容：インストール・初期設定 / APIキー・接続確認 / 「話しかけて動く」まで
- ゴール：環境構築完了。チャットで指示が通ることを体験する。
- 宿題：環境構築のメモ作成 / 簡単なタスクを1つ依頼して結果を共有

**Week 2：メモリ構築（Obsidian × Cursor）**
- テーマ：「第二の脳」をObsidianとCursorで作る
- 内容：Obsidianの基本 / マークダウンでメモ・見出し・リンク / Cursorでの編集・検索 / メモリの設計思想
- ゴール：AIに渡す"材料"を貯める土台を作る。
- 宿題：自分の事業・強み・判断基準を1本の .md にまとめる

**Week 3：Memory Systems作成（OpenClaw編）**
- テーマ：OpenClaw側でメモリを読み込ませる
- 内容：OpenClawのメモリ機能（memory.md 等） / 参照ファイル・フォルダの設計 / Memory Systemsの作成と動作確認
- ゴール：OpenClawが「自分の判断基準」を理解して動く状態にする。
- 宿題：自分用のMemory Systemを1つ作り、挙動を確認して報告

**Week 4：知識インプットDAY（GitHub × Vercel）**
- テーマ：開発・連携に必要な基礎知識を一気にインプット
- 内容：GitHubの基礎（リポジトリ・Push・branch） / Vercelの基礎（デプロイ・URL公開） / OpenClaw・Cursorとの連携イメージ / 「コードを書かずに作る」感覚
- ゴール：「開発」「連携」をチャットで指示できる土台を作る。
- 宿題：簡単なLPやページを「作ってVercelに上げて」と依頼してURLを提出

**Week 5：情報の一元管理（GitHub Actions）**
- テーマ：情報を1か所に集め、自動で更新される仕組みを作る
- 内容：情報の一元管理の考え方 / GitHub Actionsでデータ自動収集 / 収集データの保存形式 / OpenClawが参照する"正解の本"の更新
- ゴール：バラバラのデータを一元化し、AIが参照しやすい形にする。
- 宿題：集めたいデータを1種類決め、収集フローを設計して共有

**Week 6：マーケティング活用フル講義（X / note / メルマガ / LP）**
- テーマ：コンテンツ・集客をOpenClawで効率化する
- 内容：X運用の自動化（バズ分析→投稿案生成） / note・メルマガの下書き・構成案 / LP作成（チャットで指示→Vercelで公開） / 集客導線の設計（X→セミナー→サービス）
- ゴール：X/note/メルマガ/LPが「作業」じゃなくて「設計」になる。
- 宿題：自分の事業向けに「1本のX投稿案」または「LP1枚」をOpenClawに作らせて提出

**Week 7：業務自動化（チャットで開発）**
- テーマ：自分の事業の"面倒な業務"をチャットで開発して自動化する
- 内容：業務自動化の設計（何を誰がやるか） / 領収書整理・経費まわり / 日報の自動作成 / メール返信の下書き・分類 / 既存ツール（LINE・Slack等）との連携
- ゴール：領収書・日報・メール返信のうち、1つ以上を「OpenClawに任せる」形にする。
- 宿題：「自動化したい業務」を1つ選び、指示文と結果を共有

**Week 8：AI従業員の設計＋成果報告**
- テーマ：複数台・役割分担で「AI従業員軍団」を設計し、8週間の成果を共有する
- 内容：役割別AI従業員の設計（営業・リサーチ・経理・マーケ） / 情報の一元管理と複数OpenClawの連携 / 成果報告：OpenClawで作ったスライド構成で成果を共有
- ゴール：役割別にAIを割り振る設計ができるようになる。受講者同士で成果を共有。

**デザイン**
- 4ステップは #D73939 の番号で縦フロー。矢印は赤の細線
- 各Weekカードは #F5F5F5 背景・左に #D73939 の太いボーダーライン（4px）

## コピーライティングのルール

1. **事実と断定で書く**。推測・願望・曖昧な表現は使わない。
2. **数字を多用する**。「63,377人」「934人」「30%」「8週間」「60分×3回」など具体的な数字は説得力の源泉。
3. **1文は短く**。1文50文字以内を目安。句読点で切る。
4. **面談で聞いた内容を"再確認"するトーン**。押し売りではなく、整理してあげる姿勢。
5. **「何が手に入るか」を具体的に書く**。抽象的なベネフィットは避ける。
6. **プランの違いを明確にする**。「ベーシック vs プレミアム」「一般 vs ユニコスクール生」の2軸が一目でわかるように。
7. **絵文字は一切使わない**。
8. **敬体ベース**（です・ます調）。見出し・キャッチコピーは体言止め・断定の常体も可。

## 技術的な指定

- **HTML + CSS（1ファイル）** で出力する。外部CSSフレームワーク不要
- **レスポンシブ対応**（max-width: 820px、モバイルはpadding 20px）
- **Google Fonts**：Noto Sans JP（400, 700）を読み込む
- **OGP meta タグ**を含める（タイトル：「OpenClaw Workshop ── プラン選択・お申し込み」）
- **ページ速度重視**。画像は不要。SVGの装飾線程度にとどめる
- **スクロール追従のCTAバー**（モバイル時）：画面下部に「プランを選んで申し込む ↓」の固定バー。タップするとセクション12（最終CTA）までスクロール。#D73939 背景・白文字
- **タブ切り替えUI**（セクション6・12）：「一般の方」「ユニコスクール生の方」の2タブ。JSで切り替え。タブのアクティブ状態は #D73939 下線
- **外部リンク**（MOSHのURL）はすべて `target="_blank"` で新しいタブに開く

## 全体のトーン

このLPは「売り込み」ではなく「面談の内容を整理してあげるページ」です。
読者はすでに面談で全体像を聞いており、感情的にはほぼ決まっている状態。
LPの役割は「最後の整理と確認」。
だから煽らない。淡々と、でも力強く、事実を並べる。
プランの違いが明確にわかり、自分に合ったボタンを迷わず押せる設計にしてください。
