# ローンチ・ボルト運用ガイド（7_ローンチ）

このノートは **AirCle のローンチ関連 Vault の正本** です。Cursor の Skill `launch-vault-filing` もここを参照します。

---

## 1. `7_ローンチ` 直下の並び（固定イメージ）

| フォルダ | 役割 |
| --- | --- |
| **00_Skills** | 運用ルール・テンプレ（いま読んでいる場所） |
| **01_無料セミナー_OpenClaw5DAYS講座** | 5DAYS 各回フォルダ＋講座横断 |
| **02_有料スクール_8週間ワークショップ** | ステップ配信・Discord・個別相談・LP など |
| **03_スクール設計_商品・法務** | 商品概要・特商法 |
| **04_決済・インフラ** | 決済まわりの設定メモ |

番号は **「上から読む順」** と **「無料 → 有料 → 設計 → インフラ」** が一目でわかるようにしてある。

---

## 2. 各回セッション（例：`第5回_OpenClaw総決算`）の中身

**第N回のフォルダの直下**は、次の番号で統一する。

| 番号 | フォルダ名 | 何を入れるか |
| --- | --- | --- |
| 00 | `00_セミナー概要` | コンセプト、ターゲット、タイトル案、長めの台本・スライド構成案 |
| 01 | `01_告知関連` | X / 公式LINE / オープンチャット / メルマガ / Peatix など **チャネル別** |
| 02 | `02_スライド` | **スライド・口頭用の素材**（構成案、パート別原稿、スライド作成用情報など） |
| 03 | `03_ミーティング` | **打ち合わせ・MTG 1回ごとに1ノート**（決定事項・宿題・次回論点） |
| 04 | `04_次回以降のアクション` | MTG で出た **タスクの置き場**、フォロー、分析レポート、タスク管理、**申込・参加などの数字** |
| 05 | `05_講座内容` | （使う回だけ）当日の講義コンテンツ本体 |

### 廃止したもの

- **`02_運営関連`** … 中身は **`02_スライド`**（資料系）か **`04_次回以降のアクション`**（タスク系）に寄せる。

---

## 3. ミーティングが終わったら（Skill でも従う手順）

1. **`03_ミーティング`** に新規ノートを作る。  
   - 推奨ファイル名: `YYYY-MM-DD_短いテーマ.md`  
   - 中身の最小セット: **日時・参加者・決定・宿題・次回までの論点**
2. 実行タスクは **`04_次回以降のアクション`** に書く（またはミーティングノートからリンクだけ貼る）。
3. スライドや構成の変更が必要なら **`02_スライド`** または **`00_セミナー概要`** を更新する。

## 4. よくある振り分け

| 迷うもの | 置き場所 |
| --- | --- |
| X ポスト文案 | `01_告知関連/X投稿` |
| スライドの骨子・読み原稿 | `02_スライド` または `00_セミナー概要` |
| チーム MTG のメモ | `03_ミーティング` |
| 「あとでやる」一覧・締切付きタスク | `04_次回以降のアクション` |
| 申込・参加の数字 | `04_次回以降のアクション`（例: `申込者数・参加者数.md`） |

## 5. 有料スクールの LINE ステップ配信（横断）

`02_有料スクール_8週間ワークショップ/01_LINE_ステップ配信・申込後/` の **番号フォルダの意味** と **Mermaid フロー** は、別紙 **[[有料スクール_LINEステップ配信_フロー]]** にまとめてある。

**有料スクールローンチを回すたびに手を入れる箇所（最低限）**

- `02_無料5DAYS終了後_審査面談申込まで/` 配下に **`第N回_◯◯/`** を用意（`_テンプレ_新規回追加用` の複製でよい）。
- その中の **`開催中_Day3`〜`Day5`** … 無料5DAYSの該当日に合わせた「有料ワークショップ詳細」の公式LINE文案。
- 同フォルダの **`A-1`** … 最終日当日 or 翌朝のお礼〜面談案内（回ごとに中身が変わる）。
- **`00_全回共通/A-2〜A-6`** … 最終日**後**の追い配信。原則共通利用し、URL・予約状況の表現だけ更新。

---

## 6. 他プロジェクトへの横展開

別ローンチを新設するときも、**同じ番号体系**（00〜05）をコピーして使うと、検索と Cursor の整理が安定する。
