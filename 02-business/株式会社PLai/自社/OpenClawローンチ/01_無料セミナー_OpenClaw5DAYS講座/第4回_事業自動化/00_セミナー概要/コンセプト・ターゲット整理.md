# コンセプト・ターゲット整理

## セミナーのコンセプト
「OpenClaw vs Claude Code Channels」── なぜ非エンジニアにとってOpenClawが重要なのか？を徹底比較＋実演で証明する120分

### 核心メッセージ
- **OpenClawはお金稼げる魔法のツールではない**。だが、学ぶことは大きすぎる「**複利**」になる
- **ノウハウコレクターになるな**。ツールを知るだけでなく「使いこなす」ところまで行くことが重要
- **どうせAGIの未来は来る**。そのときに重要なのは「どう使いこなすか」
- Claude Code Channelsとの比較で、OpenClawの独自の強み（**永続メモリ・自律性・セルフホスト**）を明確にする
- OpenClawを使いこなすには**3つ**必要：①良質なコンテキスト ②AIとの会話術 ③運用メソッド
- OpenClawだけ学んでも意味がない → **AIの基礎力（言語化力・AIとの対話力）＋最低限の開発知識**が必要

### 戦略的位置づけ
- Day 4は「OpenClaw vs Claude Code Channels」を軸に、**いち × ユニコの120分コラボ構成**
- いちパート①（20分）で「なぜOpenClawなのか」の論理を組み立て → ユニコパート①（40分）で実演・事例で証明 → ユニコパート②（40分）でスクール紹介・本質的に学ぶ必要性を訴求 → いちパート②（20分）で未来・ワークショップに接続
- **ユニコ氏が約80分**を担当し、実演のスピード感とスクールの説得力で**特別感**を最大化する
- 単なるツール比較で終わらず、「AIの基礎力がないとOpenClawも使いこなせない」→ だからユニコスクールが必要、という導線を自然に作る
- 「ノウハウコレクター問題」を提起し、学びを行動に変える仕組みとしてワークショップ・スクールを位置づける

## ターゲット層

### プライマリターゲット
- **OpenClawに興味があるが、Claude Code Channelsとの違いがわからない人**
  - 「どっちを使えばいいの？」が解決できていない
  - ツールの表面的な機能比較ではなく、本質的な違いを知りたい

- **AIツールを学んでいるが「ノウハウコレクター」状態の人**
  - いろいろ触ってみたが、実際に使いこなせていない
  - 学んだことを事業や仕事に活かせていない

- **非エンジニアでAI開発ツールを活用したい人**
  - Claude CodeやCursorは聞いたことがあるが、ハードルが高い
  - チャットベースで開発・自動化できるツールを探している

### セカンダリターゲット
- **OpenClaw導入済みで活用度を上げたい人**
  - 第1〜3回からの継続参加者
  - コンテキスト設計・会話術・運用メソッドを体系的に学びたい
- **AGI時代に備えて「使いこなす力」を身につけたい人**
  - ツールの使い方だけでなく、AIとの付き合い方を本質的に理解したい

## ペインポイント
1. OpenClawとClaude Code Channelsの違いがわからず、どちらを選べばいいか判断できない
2. AIツールを色々触っているが「ノウハウコレクター」で終わっている
3. OpenClawを導入したが、使いこなせている実感がない
4. 非エンジニアだが、AI開発ツールのハードルが高くて始められない
5. AGI時代に何を準備すべきかわからない

## ソリューション
1. **OpenClaw vs Claude Code Channelsを徹底比較**：永続メモリ・自律性・セルフホストの3つの独自価値を具体的に示す
2. **ユニコによる実演デモ**：OpenClawで実際に何ができるかをスピード感あるデモで証明する
3. **「使いこなす」ための3条件を提示**：①良質なコンテキスト ②AIとの会話術 ③運用メソッド
4. **AIの基礎力の重要性を説く**：OpenClawだけ学んでも意味がない → 言語化力・対話力・開発知識が必要 → ユニコスクールが選択肢
5. **ワークショップで実践の場を提供**：学びを行動に変える仕組みとしてOpenClawワークショップを案内
