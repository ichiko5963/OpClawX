# 02 Save for Later / 保存版型

## 🇺🇸 English

### Definition
Content that provides "bookmark-worthy" value. Triggers the desire to save for future reference and FOMO.

### Psychology
- **Future Reference**: Want to save for later
- **FOMO**: Fear of losing valuable information
- **Utility**: Practical value recognition

### Structure
```
【Save for Later】+ Who + What + Guide 📌
↓
Why it's important (1 sentence)
↓
3-5 specific methods (bullet points)
↓
Call to action
```

### Prompt Template
```
Create a "Save for Later" style post about {theme}:
- Start with 【保存版】
- Include specific numbers/tools
- 3-5 bullet points of methods
- Use 📌 and 👇 only
- 200-280 characters
```

### Example
> 【保存版】$300k ARR with Vibe Coding - Alex Finn's complete guide 📌 The 3 steps I actually use 👇 ① Niche selection (low competition) ② Automation tools ③ Continuous improvement. Results after 3 months 👇

## 🇯🇵 日本語

### 定義
「後で見返したい」価値を提供するコンテンツ。後で見返したいという安心感とFOMOを刺激。

### 心理メカニズム
- **将来の参照**: 後で見返したい
- **FOMO**: 価値ある情報を逃したくない
- **実用性**: 実際に役立つと認識

### 構造
```
【保存版】+ 誰が + 何を + ガイド 📌
↓
なぜ重要か（1文）
↓
具体的手法3-5個（箇条書き）
↓
実践を促す
```

### プロンプトテンプレート
```
{theme}について保存版型の投稿を作成:
- 【保存版】から始める
- 具体的な数字・ツール名を含める
- 手法を3-5個箇条書き
- 📌と👇のみ使用
- 200-280文字
```

### 例文
> 【保存版】Vibeコーディングで月10万円稼ぐ方法📌 僕が実際にやっている3つのステップ👇 ①ニッチ選定（競合少ない市場）②自動化ツールの導入 ③継続的な改善。これを3ヶ月続けた結果。詳細は👇

### 💡 コツ
- 「普通なら有料」「◯◯円相当」で価値を強調
- 具体的手順より「何をすればいいか」を簡潔に
- 「後で見返す」「保存推奨」などの言及を含める
