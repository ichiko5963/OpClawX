# 04 結論型 - Conclusion First

## 🎯 定義

結論を最初に提示し、理由を後から説明する型。
読者の「結論が知りたい」欲求を即座に満たす。

## 🧠 心理メカニズム

1. **即時満足**: 結論を先に知りたい
2. **認知負荷軽減**: 結論がわかると読みやすくなる
3. **衝撃の伝播**: 意外な結論は拡散される

## 📝 構文の骨格

```
【結論】+ 核心メッセージ（衝撃的な主張）💡
↓
その理由を3点で説明
↓
これが意味すること
↓
実践の呼びかけ👇
```

## 🎨 生成プロンプト

```
以下のテーマで、SNSでバズる【結論型】の投稿を作成してください：

【条件】
- テーマ: {theme}
- 最初に「結論：」で核心を述べる
- その理由を3点で簡潔に
- 「これが意味すること」で解説
- 絵文字は💡👇のみ使用
- 文字数: 200-280文字
```

## 💡 コツ

- 結論は「意外性」または「有用性」のどちらかを持たせる
- 「〜すべき」「〜の時代」などの明確な主張で締める
- 理由は簡潔に、核心のみを伝える
- 「みんなが思ってること」を言うと共感が生まれる

## ✅ 代表投稿例

> 【結論】Manus 1.6 は業務支援ツールではなく、AIオーケストレーション基盤だ。GA開始で主導権争いが加速。追う側と追われる側の差が歴然。これはAI業界の構造変化の始まり。詳細は👇
