# 06-学び ルール — **最重要フォルダ**

## 目的

日々の経験から抽出した学びを、カテゴリ別に蓄積する。
具体的な出来事 → 抽象化した原則（ただし抽象の中でも具体的に書く）。

## 構造

```
06-学び/
├── _RULE.md           ← このファイル
├── _WEEKLY-LOG.md     ← 週次レビュー実行記録
├── [カテゴリ名]/
│   ├── insight.md     ← 抽象化された学び（本質）
│   └── episodes/
│       └── YYYY-MM-DD-[タイトル].md  ← 具体エピソード
```

## 週次自動蓄積ルール

毎週（日曜日）、以下を自動実行：

1. 直近1週間の `03-daily/00-daily-note/` を全て読む
2. 起きた出来事・結果を踏まえて学びを抽出
3. 既存カテゴリに該当するエピソードを追加
4. **毎週3個の新しい学びカテゴリを作成**
5. `_WEEKLY-LOG.md` に実行記録を追記

## カテゴリの例

- `人を誘うときは相手の好きを理由に誘え/`
- `自分の欲望を優先することは最も稚拙なこと/`
- `人付き合いがうまい人は理由付けがうまい/`

## insight.md のフォーマット

```markdown
# [カテゴリ名]

## 本質
[2-3文で核心を書く]

## なぜ大事か
[この学びが重要な理由]

## 実践方法
[具体的にどう使うか]
```

## episodes/ のフォーマット

```markdown
# YYYY-MM-DD [タイトル]

## 何が起きたか
[時系列で事実を記載]

## 結果
[どうなったか]

## この経験から得た学び
[insight.md の内容と紐づけ]
```

## コンテンツ作成への活用

メルマガ・note・X投稿で体験談を使いたい場合：
→ `06-学び/` のカテゴリから関連する学びを検索
→ `episodes/` の具体エピソードを引用
→ `insight.md` の本質を軸にコンテンツを構成

## 05-input との連携

- Kindle/YouTube/インサイドストーリーズの内容も、自分の体験と結びつけて学びカテゴリに入れる
- 本の内容だけのメモは `05-input/` に、自分の体験と結びついた学びは `06-学び/` に
