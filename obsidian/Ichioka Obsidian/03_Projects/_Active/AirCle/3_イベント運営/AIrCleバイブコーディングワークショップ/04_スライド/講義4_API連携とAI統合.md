# 講義4: API連携とAI統合

## 世界中のデータを取り込み、アプリに「知能」を宿す

---

# 今日できるようになること

1. 🔌 **API** の仕組みを理解し、天気データを取り込む
2. 🧠 **Gemini API** を組み込み、アプリにAI機能を追加する
3. 🗺️ **バイブコーディングの全ワークフロー**をマスターする

---

# レストランで例えると？

- **客（あなたのアプリ）**: 「ハンバーグ（天気データ）ください」
- **厨房（サーバー）**: 実際にデータを作っている場所。客は立ち入り禁止。
- **ウェイター（API）**: 客の注文を聞いて厨房に伝え、料理を持って帰ってくる人。
- **APIがないと、厨房に入れないのでデータがもらえない。**

---

# 登録不要の神API

- 世の中のAPIは「登録（API Key）」が必要なものが多い（面倒くさい）。
- **Open-Meteo** は、登録不要・無料で天気データをくれる神サービス。
- 練習にはこれが最適！

---

# 🛠️ 直叩きしてみる

1. ブラウザの新しいタブを開く。
2. アドレスバーに `https://api.open-meteo.com/v1/forecast?latitude=35.689&longitude=139.691&current_weather=true` と入力してEnter。
3. 画面に文字（JSON）がブワッと出たら成功！

---

# 🛠️ 部品の発注

**指示（Antigravity）**: **「`Weather.jsx` という新しいコンポーネントを作って。 Open-MeteoのAPIを使って、東京の現在の天気と気温を取得して表示して。 デザインはシンプルに、晴れなら太陽、雨なら傘のアイコンを出して。」**

---

# ただ表示するだけじゃつまらない

- 天気を表示するだけなら、天気予報アプリを見ればいい。
- **「天気によって、Todoアプリの挙動を変える」** のが面白い。
- 例：雨なら「傘を持っていく」というタスクを自動追加する。

---

# 鍵は玄関マットの下に置くな

- API Keyをコードに直接書くと、GitHubで公開された時に**全世界に流出**する。
- **`.env` ファイル** に書いて、Gitにはアップしないのが鉄則。
- AIに聞こう：**「API Keyを安全に管理する方法を教えて」**

---

# アプリに「脳」を移植する

- 天気APIは「事実を返すだけ」。
- **Gemini API** は「考えて、生成する」。
- 違い：決まった答えがない。文脈を理解する。

---

# 脳みその調達場所

1. `aistudio.google.com` にアクセス（Googleアカウントでログイン）。
2. **「Get API key」** をクリック。
3. **「Create API key」** でキーを発行し、コピーする。

---

# 🛠️ 相談窓口を作る

**指示**: **「タスクリストの下に『🔥 修造に相談する』というボタンを作って。 押すと、現在のタスク一覧を読み取って、Geminiに送信する関数も作って。 プロンプトは『あなたは熱血コーチです。以下のタスクリストを見て、ユーザーを全力で励まし、具体的なアドバイスを1つください』として。」**

---

# AIに枠をはめる

- 励ましも嬉しいけど、**「タスクを分解」** してくれたらもっと便利。
- 例：「カレーを作る」→ 「肉を買う」「野菜を切る」「煮込む」に分解して、**自動でリストに追加**してほしい。
- プロンプトの最後に： **「回答は必ずJSON形式にしてください。例：`[{"task": "野菜を切る"}, ...]`」**

---

# The Vibe Coding Flow

1. **Start**: エディタ（Antigravity）を開く
2. **Base**: Next.jsで土台作成（UI・ルーティング）
3. **DB**: Supabaseで認証・CRUD確保
4. **Plan**: 要件定義（誰の・何の課題？）
5. **Research**: 実現可能性の調査
6. **Ticket**: **タスク分解プロンプト**（ここが重要！）
7. **Build**: チケット消化（実装と確認の反復）
8. **Finish**: テスト・デプロイ準備

---

# ★今日の最重要テクニック

**プロンプト例**:
```
決定した要件を元に、実装ステップを「細かいタスクリスト」に分解してください。
1つのタスクは「1ファイル修正」または「1機能実装」レベルまで小さくすること。
【フォーマット】
- [ ] Task 1: 〇〇画面のUI作成
- [ ] Task 2: Supabaseへの接続設定
- [ ] Task 3: ...
このリストを常に記憶し、私が「次」と言ったら、次のタスクを実行してください。
```

---

# ひたすら消化する

1. AI：「次は Task 1: UI作成 です。実行しますか？」
2. あなた：「やって」
3. AIがコードを書く → プレビュー確認。
4. あなた：「OK、次」または「ここ直して」
5. **リストの `[ ]` が `[x]` に変わっていく快感！**

---

# 記憶を保たせる

- AIは前の話を忘れがち。
- 毎回、**「Step 4の要件定義」と「Step 6のタスクリスト」を参照（メンション）** させながら指示する。
- 「今の実装、要件とズレてない？」と定期的にチェックさせる。
