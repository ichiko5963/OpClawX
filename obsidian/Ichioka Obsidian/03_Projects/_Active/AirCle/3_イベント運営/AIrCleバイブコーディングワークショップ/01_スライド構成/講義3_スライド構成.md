## 【Part 2: Webの基礎（スライド4-8）】

### スライド4: HTML（骨）
**タイトル**: タグ（荷札）による構造化  
**内容**:
- `<h1>` = 「これは大見出しです」という荷札
- `<p>` = 「これは段落です」という荷札
- `<button>` = 「これはボタンです」という荷札
- コンピュータに「これは何なのか」を教えるためのマーク  
**発表者ノート**: 見た目を変えるためにHTMLを使うのは間違いです。HTMLはあくまで「意味」を定義するためにあります。

### スライド5: CSS（服）
**タイトル**: 指定のルール  
**内容**:
- 「どこの（セレクタ）」
- 「何を（プロパティ）」
- 「どうする（値）」
- 例：`h1 { color: red; }` ➡ 「大見出しの、色を、赤にする」  
**発表者ノート**: v0が書いている `class="bg-red-500"` みたいなやつ（Tailwind）も、結局はCSSの省略記法です。

### スライド6: JavaScript（筋肉）
**タイトル**: イベント駆動（Event Driven）  
**内容**:
- 常に「待ち構えている」。
- 「クリックされたら（イベント）」➡「計算する（処理）」
- ページを書き換えたり、通信したり、何でもできる万能細胞。  
**発表者ノート**: 「おみくじ」を思い出してください。あれがJavaScriptの正体です。

### スライド7: React（レゴブロック）
**タイトル**: 全ては「部品（コンポーネント）」  
**内容**:
- 画面を小さな「部品」に分ける。
- 「ボタン部品」「ヘッダー部品」「カード部品」。
- 部品を組み合わせるだけでアプリができる。
- **AIは、この「部品作り」がめちゃくちゃ得意。**  
**発表者ノート**: バイブコーディングでReactを使う理由はこれです。AIに「ボタン部品作って」と頼みやすいからです。

### スライド8: Next.js（完全装備セット）
**タイトル**: Reactの「完全装備セット」  
**内容**:
- Reactは「部品を作る技術」。
- **Next.js** は、Reactを使って「Webサイト全体（ページ移動や高速化）」を作るためのフレームワーク。
- Antigravityのテンプレートもこれ。  
**発表者ノート**: Reactが「エンジン」なら、Next.jsは「車体」です。私たちはNext.jsという車に乗って開発します。

## 【Part 3: データベース（スライド9-14）】

### スライド9: データの3段階
**タイトル**: 記憶の進化  
**内容**:
1. **State（変数）**: 手のひら（リロードで消える）
2. **LocalStorage**: ポケット（リロードで消えないが、**端末が変わると無い**）
3. **Cloud DB**: 銀行（ネット上にある。**どこからでも見れる**）  
**発表者ノート**: まずは「ポケット（LocalStorage）」から。一番簡単だからです。

### スライド10: Supabaseとは？
**タイトル**: 空飛ぶExcel  
**内容**:
- 難しくない。**「ネット上にExcelファイルがある」** と思えばいい。
- **Table（テーブル）**: Excelの「シート」
- **Column（カラム）**: 列（タスク名、完了フラグなど）
- アプリから、このExcelに書き込みに行くだけ。  
**発表者ノート**: 「DB構築」とか言うと怖いですが、「Excelの表を作る」だけです。

### スライド11: 【演習】Supabaseの準備
**タイトル**: 🛠️ 銀行口座を作る  
**手順**:
1. `supabase.com` にアクセス → GitHubアカウントでログイン
2. **「New Project」** をクリック
3. 名前：`my-todo-app`、パスワードを設定
4. **「Create Project」**（数分でデータベースが爆誕）  
**発表者ノート**: これであなたは、世界中にサーバーを持つデータベースの管理者になりました。

### スライド12: 【演習】テーブル作成（AIに頼む）
**タイトル**: 🛠️ Excelの表を作る  
**内容**:
- Supabaseには **「AI SQL Editor」** がある。
- チャット欄に指示： **「Todoアプリ用のテーブルを作って。タスク名(text)と、完了フラグ(boolean)と、作成日時(timestamp)が必要。」**
- 実行ボタンを押すと、テーブルができる。  
**発表者ノート**: SQLという言語が必要ですが、AIが書くので覚えなくていいです。

### スライド13: Antigravityと接続
**タイトル**: 🛠️ 合言葉をコピーせよ  
**内容**:
- AntigravityからSupabaseにアクセスするには「鍵」が必要。
- Supabaseの `Project Settings` -> `API` を開く。
- **URL** と **anon key** の2つをコピーする。
- `.env` ファイルに設定する。  
**発表者ノート**: これが「銀行の暗証番号」みたいなものです。

### スライド14: データの保存・読み込み
**タイトル**: 🛠️ 銀行に書き込む  
**内容**:
- Antigravityのエージェントに指示： **「LocalStorageはやめる。Supabaseを使って、データの保存・読み込みをするようにコードを書き換えて。」**
- エージェントが `supabase-js` をインストールし、コードを総入れ替えする。
- 動作確認：PCでタスクを追加 → スマホで確認  
**発表者ノート**: ここがハイライトです。保存先を「ポケット」から「銀行」に切り替えます。

## 【Part 4: 認証とセキュリティ（スライド15-19）】

### スライド15: なぜ認証が必要？
**タイトル**: 「みんなの広場」から「自分だけの個室」へ  
**内容**:
- 今の状態だと、**世界中の全員が同じTodoリストを見ている状態**（公開掲示板）。
- **「自分だけのデータ」** にするために、ログイン機能をつける。
- Supabaseなら「Googleログイン」も一瞬で実装可能！  
**発表者ノート**: 今は「みんなの広場」にデータを置いてます。次は、そこに「鍵付きの個室」を作ります。

### スライド16: Google Authの設定
**タイトル**: 🛠️ 受付カウンター設置  
**手順**:
1. Supabaseの左メニュー `Authentication` → `Providers`
2. **`Google`** を選択
3. **`Enable Google provider`**: ✅ ON
4. Google Cloud ConsoleでOAuth 2.0のクライアントID・シークレットを取得し、Supabaseに貼り付け
5. **`Site URL`** にAntigravityのプレビューURLを設定  
**発表者ノート**: Googleアカウントでのログインが一般的で、メール確認の手間もありません。

### スライド17: ログイン画面を作る
**タイトル**: 🛠️ 玄関を作る  
**指示（Antigravity）**: **「ログイン画面（Login.jsx）を作って。Supabaseの `signInWithOAuth` を使って、Googleでログインするボタンを配置して。クリックしたらGoogleの認証画面に遷移し、認証後にアプリに戻るようにして。」**  
**発表者ノート**: ボタン1つでGoogleアカウント選択→ログイン完了です。

### スライド18: RLS（セキュリティ）の設定
**タイトル**: 🛠️ 結界を張る  
**内容**:
- ログインはできた。でも、データはまだ「みんなの広場」にある。
- **RLS（Row Level Security）**: データベース側でブロックする仕組み。
- ルール：「IDカード（user_id）が一致する人しか、その行を見ちゃダメ！」
- AIに指示： **「`user_id` が自分のIDと一致する行だけ、SELECT/INSERT/DELETE できるポリシーを作って」**  
**発表者ノート**: SQLが書けなくても大丈夫。AIアシスタントに日本語で頼めば、完璧なセキュリティルールを作ってくれます。

### スライド19: 動作確認
**タイトル**: 🛠️ 侵入実験  
**手順**:
1. 普通のブラウザで「Aさん」としてタスクを書く。
2. **シークレットモード** を開く。
3. 「Bさん（別のメアド）」としてログインする。
4. **Aさんのタスクが1つも見えなければ成功！**  
**発表者ノート**: 世界が分断されました。これで、誰に見せても恥ずかしくない安全なアプリになりました。
