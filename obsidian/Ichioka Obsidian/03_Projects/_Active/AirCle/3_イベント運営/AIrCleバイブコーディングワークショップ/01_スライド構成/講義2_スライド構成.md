## 【Part 2: v0で神デザインを作る（スライド4-8）】

### スライド4: v0とは？
**タイトル**: Vercel社が生んだ「Generative UI」  
**内容**:
- 画像生成AI（Midjourneyなど）の「Webサイト版」
- **「こんな画面作って」と言うと、絵ではなく「コード」で画面を作ってくれる**
- 作られるのは、現代最強の技術セット（React + Tailwind CSS）  
**発表者ノート**: v0はただのお絵かきツールではありません。「実際に動く部品」を作ってくれます。

### スライド5: 【演習】v0のアカウント作成
**タイトル**: 🛠️ v0の世界へ  
**手順**:
1. `v0.dev` にアクセス
2. **「Log in」** をクリック（Vercelアカウントが必要）
3. 無料プランで始められる  
**発表者ノート**: まずはログインしましょう。Vercel（バーセル）という会社のアカウントを使います。

### スライド6: 最初のプロンプト
**タイトル**: 🛠️ 言葉を形にする  
**プロンプト例**:
```
iPhoneの計算機アプリのような、モダンでミニマルな電卓を作って。
ボタンは丸く、配色は黒とオレンジで。
```  
**発表者ノート**: Geminiの時と同じです。具体的であればあるほど良いですが、「モダンな」「ミニマルな」といった雰囲気（バイブス）の指示がよく通じます。

### スライド7: 「3つの提案」から選ぶ
**タイトル**: A案、B案、C案  
**内容**:
- v0は一度に3種類のデザイン案を出してくれる
- あなたの仕事は **「選ぶ（Curating）」** こと
- 「B案の雰囲気が好きだな」→ Bをクリック  
**発表者ノート**: プロのデザイナーでも3案出すのは大変ですが、v0は10秒でやります。

### スライド8: チャットで修正指示
**タイトル**: 🛠️ もっとワガママに  
**内容**:
- 選んだ案に対して、追加で指示を出す
- **「背景をダークモードにして」**
- **「数字をもっと大きく、太くして」**
- **「ボタンに影（シャドウ）をつけて立体的にして」**  
**発表者ノート**: 一度で完璧じゃなくていいんです。「ここをこうして」と会話しながら理想に近づけていきます。

## 【Part 3: Antigravityで開発環境を整える（スライド9-13）】

### スライド9: VS Codeって何？
**タイトル**: 正体は「メモ帳」  
**内容**:
- 文章を書くソフト ➡ **Word**
- 表計算をするソフト ➡ **Excel**
- プログラム（コード）を書くソフト ➡ **VS Code**
- **これだけの違いです。**  
**発表者ノート**: Wordで小説を書くように、VS Codeでアプリを書くだけです。爆発したりしません。

### スライド10: Antigravityの正体
**タイトル**: VS Code + AI  
**内容**:
- **VS Code**: 便利なメモ帳。
- **Antigravity**: そのメモ帳に、**GoogleのAI（Gemini）が最初から住んでいる**。
- 自分で設定しなくても、最初からAIが手伝ってくれる特別版です。  
**発表者ノート**: 「AI付きの高機能メモ帳」を、Googleがパッケージして配ってくれているのがAntigravityです。

### スライド11: 【演習】Antigravityのインストール
**タイトル**: 🛠️ 入手する  
**手順**:
1. `antigravity.google` にアクセス
2. **「Download for Desktop」** をクリック
3. インストーラーを起動  
**発表者ノート**: 普通のソフトと同じです。ダウンロードしてポチポチ進めるだけ。

### スライド12: 画面の見方（3分割）
**タイトル**: コックピットの構造  
**内容**:
1. **左側（サイドバー）**: **「フォルダの中身」**。PCのファイル一覧と同じ。
2. **真ん中（エディタ）**: **「書く場所」**。ここに文字を打つ。
3. **下側（ターミナル）**: **「命令する場所」**。AIやPCへのチャット欄。  
**発表者ノート**: 画面がいっぱいあって怖そうですが、見るのは「真ん中」だけです。左は「ファイル選ぶ場所」、下は「AIへのLINE」です。

### スライド13: AIエージェントを使う
**タイトル**: 執事を呼び出す  
**内容**:
- 画面右側のパネルを開く。
- ここが **「AIへの指示出しチャット」** です。
- v0で作ったデザインのコードをコピーして、ここに貼り付けて「これで作って」と頼む  
**発表者ノート**: 真ん中の画面（エディタ）で文字を打つのは、あなたじゃなくてAIです。あなたはこの右側のチャットを使います。

## 【Part 4: ターミナルとAIツール（スライド14-18）】

### スライド14: ターミナルって何？
**タイトル**: パソコンとの「LINE」です  
**内容**:
- **GUI（マウス操作）**: レストランでメニューを指差して注文する。「これください」
- **CLI（ターミナル）**: 厨房のシェフに直接LINEする。「メニューにないけど、豚肉を焼いて塩コショウして！」
- **ターミナル = パソコンへの「直通チャット」**  
**発表者ノート**: 黒い画面は怖くありません。ただのチャットアプリです。相手が人間じゃなくてパソコンなだけ。

### スライド15: なぜターミナルを使うの？
**タイトル**: マウスより「言葉」が強い  
**内容**:
- マウスだと「ファイルを作って、名前を変えて、移動して...」と3回操作が必要。
- ターミナルなら **「これやっといて」の一言（1行）** で終わる。
- **特にAIとの相性が抜群！**（AIは言葉が得意だから）  
**発表者ノート**: AI時代になって、ターミナルの価値が爆上がりしました。

### スライド16: 3大AIツールの使い分け
**タイトル**: どれを使えばいいの？  
**内容**:
- **Claude Code** = 現場監督（全体構築）
- **Codex** = 外科医（修正・エラー）
- **Gemini CLI** = 巨人（大量データ）  
**発表者ノート**: 彼らは全員「天才」ですが、得意分野が違います。スポーツチームのようなものです。

### スライド17: 【演習】Claudeを呼んでみる
**タイトル**: 🛠️ 現場監督を召喚  
**手順**:
1. Antigravityのターミナルを開く
2. `claude` と入力してEnter
3. 「何を作りますか？」と聞かれるので、日本語で答える  
**発表者ノート**: 黒い画面に `claude` と打つだけ。それだけで、優秀な建築士が「はい、なんでしょう」と現れます。

### スライド18: 対話で作らせる
**タイトル**: 🛠️ チャット形式で進行  
**内容**:
- 黒い画面の中で、ClaudeとLINEする感覚
- あなた：「Reactで簡単な電卓アプリを作って」
- Claude：「わかりました。ファイルを作成します...完了しました」
- **マウスを使わずにアプリができた！**  
**発表者ノート**: これが「CLI（コマンドライン）」の威力です。ファイル作成、保存、フォルダ移動...全部彼がやってくれました。
