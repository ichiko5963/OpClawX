## 【Part 2: React/Next.jsの理論（20分）】

### なぜ理論が必要？

AIは優秀ですが、たまに**「嘘の呪文」**を教えます。

基礎を知らないと、その嘘に気づけず、永遠にバグが直りません。

**「理論」は、AIの間違いを指摘するための武器になります。**

---

### React = 「レゴブロック」

Reactは、画面を小さな「部品（コンポーネント）」に分けて、組み合わせる技術です。

- 「Button.tsx」というファイルを作りましたよね？あれは「ボタンというブロック」を定義していたんです。
- ヘッダー部品、ボタン部品、カード部品...
- これらを `Components` フォルダに入れておき、パズルのように組み合わせるのがReactです。

---

### State（ステート）= 「記憶」

**普通の変数**: 書き換えても、画面は変わらない（リロードが必要）。

**State (`useState`)**: **書き換えた瞬間、画面が自動で再描画される魔法の変数。**

「おみくじ結果」や「入力フォームの文字」は、全部これ。

`useState`。これは「画面を書き換えるスイッチ付きの箱」です。ここが変わると、Reactが「おっ、画面更新しなきゃ！」と反応します。

---

### Server vs Client

Next.jsには「2つの世界」があります。

**Server Component (厨房)**: サーバー側。DBに近い。安全。でも**客の操作（クリック）はできない**。

**Client Component (客席)**: ブラウザ側。**クリックや入力ができる**。でもDBに直接触るのは危険。

**`"use client"`** = 「これは厨房じゃなくて、客席（ブラウザ）で動かす部品ですよ」という宣言。

---

### 使い分けの基準

- Q. **データベースやAPIキーを使いますか？** → YES ➡ **Server Component** (デフォルト)
- Q. **`onClick` や `useState` を使いますか？** → YES ➡ **Client Component** (`"use client"` をつける)

迷ったらこれを見てください。最初は全部 `"use client"` でも動きますが、慣れてきたら使い分けるとプロっぽくなります。

## 【Part 3: GitHubでバックアップ（20分）】

### Git = 「セーブポイント」

**「Git（ギット）」とは？**

Gitは、**ゲームの「セーブポイント」**だと思ってください。

ファイルを上書き保存すると、過去の状態は消えます。

**Git** は、過去のあらゆる時点に「セーブポイント（コミット）」を作れます。

失敗したら、昨日のセーブポイントまで時間を巻き戻せます。

「バグった！終わった！」とならないためのタイムマシン。それがGitです。

---

### GitHub = 「コードの図書館」

**「GitHub（ギットハブ）」とは？**

- Gitは「自分のPC内」の履歴。
- **GitHub** は、それを「クラウド（ネット上）」に保存する場所。
- 全世界のエンジニア（やAI）がコードを公開・共有している。
- **デプロイ（Vercel）** も、ここからコードを引っ張っていく。

GitHubは「エンジニアのインスタグラム」です。自分の書いたコードをアップして、世界に見せたり、保存したりします。

---

### 【実践】最初のコミット

**【手順】初セーブ**

1. Antigravityの左メニュー「ソース管理」タブを開く。
2. 「変更」欄にファイルがいっぱいあるはず。
3. **「＋」ボタン**（変更をステージング）を押す ➡ カゴに入った状態。
4. メッセージ欄に **"First Commit"** と書く。
5. **「コミット」ボタン** を押す。

これでリストが消えましたね？ それは「全ての変更が記録された」からです。おめでとうございます、最初のセーブポイントが作られました。

---

### 【実践】GitHubにPush

**【手順】倉庫に送る**

1. `github.com` にログイン。
2. 右上の **「+」** -> **「New repository」**。
3. Repository name: `my-vibe-app`
4. Public（公開）か Private（自分だけ）を選ぶ（練習ならPublicでOK）。
5. **「Create repository」**。
6. GitHubの画面に表示されるコマンドをコピーして、Antigravityのターミナルで実行。

これであなたのコードはクラウドに保存されました。もうPCを窓から投げ捨てても大丈夫です（投げないでください）。

---

### 【超重要】環境変数の罠

GitHubを見てみましょう。**`.env` ファイルがありませんね？**

これは `.gitignore` という設定ファイルが「鍵は送るな！」と止めてくれているからです。

**絶対にAPIキーをGitHubに上げてはいけません。**（数秒で盗まれて悪用されます）

Antigravity賢いですね。危険物は自動で検知して、倉庫への配送をブロックしてくれています。

## 【Part 4: Vercelでデプロイ（25分）】

### Vercelとは？

**「Vercel（バーセル）」とは？**

Vercelは、Next.jsを開発している会社が作った、Web公開サービスです。

**特徴**:
- **設定ゼロ**：Next.jsなら自動で認識して最適化してくれる。
- **爆速**：世界中にサーバーがある。
- **無料**：個人開発なら十分すぎる無料枠。

Next.jsで作ったなら、Vercel一択です。iPhoneとMacくらい相性が抜群です。

---

### 【実践】Vercelに接続

**【手順】店を構える**

1. `vercel.com` にアクセス -> **「Sign Up」**。
2. **「Continue with GitHub」** を選ぶ（これが重要！）。
3. ログインすると、ダッシュボードが表示される。
4. **「Add New...」** -> **「Project」**。
5. **「Import Git Repository」** のリストに、前回作った `my-vibe-app` があるはず。
6. **「Import」** ボタンを押す。

倉庫にある荷物を選んで「これを店に出すよ」と指示します。まだ「Deploy」は押しちゃダメですよ！

---

### 【最重要】環境変数の設定

ここで「Deploy」を押すと、**エラーになります（または動きません）。**

なぜ？ -> **「鍵（.env）」がGitHubにないから。**

セキュリティのため、鍵は倉庫（GitHub）には置かないルールでした。

**店舗（Vercel）の金庫に、直接鍵を入れる必要があります。**

**【手順】金庫に鍵を入れる**

1. Import画面にある **「Environment Variables」** をクリックして開く。
2. Antigravityの `.env` ファイルの中身をコピペする。
   - Name: `VITE_SUPABASE_URL`, Value: `https://...`
   - Name: `VITE_GEMINI_API_KEY`, Value: `AIza...`
3. 1つずつ「Add」を押して登録する。

SupabaseやGeminiの鍵を、一つずつ丁寧に登録してください。これがないと、本番環境でデータベースに接続できません。

---

### デプロイ実行！

**【手順】スイッチオン**

1. 鍵の設定が終わったら...
2. **「Deploy」** ボタンをクリック！
3. 画面が切り替わり、ビルド（建築）が始まる。
4. 1〜2分待つ...
5. **「Congratulations!」** の文字が出たら成功！
6. サムネイル画像をクリックすると...
7. **`https://my-vibe-app.vercel.app`** のようなURLでアプリが開く！

おめでとうございます！ これがあなたのアプリの「世界共通住所」です。

---

### 実機テスト

**【手順】手のひらの上の魔法**

1. そのURLをスマホに送る（LINEやAirDropで）。
2. スマホで開く（WiFiを切って4G/5Gで）。
3. ログインして、タスクを追加してみる。
4. **PCで見ると、リアルタイムで同期されている！**

「自分のPC」という殻を破り、インターネット上で動いています。これがWebアプリ開発の醍醐味です。
