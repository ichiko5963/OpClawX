## 【Part 2: APIって何？（概念理解）】

### スライド 4: API = 「ウェイター」

**タイトル**: レストランで例えると？ **ビジュアル**: 客（あなた）、ウェイター（API）、厨房（Googleなどのサーバー）の図 **内容**:

- **客（あなたのアプリ）**: 「ハンバーグ（天気データ）ください」
    
- **厨房（サーバー）**: 実際にデータを作っている場所。客は立ち入り禁止。
    
- **ウェイター（API）**: 客の注文を聞いて厨房に伝え、料理を持って帰ってくる人。
    
- **APIがないと、厨房に入れないのでデータがもらえない。** **発表者ノート**: APIは「注文の窓口」であり「運び屋」です。このウェイターさんに正しく注文できれば、どんなデータでも手に入ります。
    

### スライド 5: JSON = 「ウェイターの言語」

**タイトル**: 注文は英語じゃない **ビジュアル**: `{ "weather": "rainy", "temp": 25 }` みたいな文字の羅列 **内容**:

- ウェイター（API）は、人間語ではなく **「JSON（ジェイソン）」** という言葉で話す。
    
- カッコ `{ }` とコロン `:` で書かれたデータの塊。
    
- 難しそうに見えるけど、**AIはこのJSONを読むのが大得意。** **発表者ノート**: 人間には読みづらいですが、AIにとっては母国語みたいなものです。「このJSONを解析して」と頼めば一瞬です。
    

### スライド 6: エンドポイント = 「メニューの場所」

**タイトル**: どこに注文する？ **内容**:

- **URL（エンドポイント）**: 注文を受け付けている窓口の住所。
    
- 例: `https://api.open-meteo.com/v1/forecast...`
    
- このURLにアクセスすることが、「注文する」ということ。 **発表者ノート**: Webサイトを見る時もURLを打ちますよね？APIも同じです。URLを叩けばデータが返ってきます。

## 【Part 5: データを「使う」（連携）】

### スライド 13: ただ表示するだけじゃつまらない

**タイトル**: データ連携の真髄 **内容**:

- 天気を表示するだけなら、天気予報アプリを見ればいい。
    
- **「天気によって、Todoアプリの挙動を変える」** のが面白い。
    
- 例：雨なら「傘を持っていく」というタスクを自動追加する。 **発表者ノート**: ここからがプログラミングの醍醐味です。「外部データ」をトリガーにして、アプリを賢くします。
    

### スライド 14: ロジックの追加

**タイトル**: 🛠️ アプリを賢くする **指示**: **「もし天気が『雨（Rain）』だったら、 タスクリストの一番上に『🌂 傘を忘れない』というタスクを自動で表示して（保存はしなくていい）。 警告色（赤っぽい背景）で目立たせて。」** **発表者ノート**: AIに条件分岐（if文）を書かせます。「雨ならこうする」というルールを教え込むのです。

### スライド 15: テスト（天気を偽装する）

**タイトル**: 🛠️ 晴れ男の悩み **内容**:

- 今日が晴れだと、雨の動きが確認できない。

- 画面に「🌂 傘を忘れない」が出たら成功！ **発表者ノート**: 開発中は嘘のデータを使ってテストします。これを「モック（模型）」と言います。

## 【Part 6: API Key（鍵）の話】

### スライド 16: 有料APIの壁

**タイトル**: ⚠️ 全てが無料ではない **内容**:

- Google Maps、ChatGPT、TwitterなどのAPIは **「API Key（会員証）」** が必要。
    
- 使った分だけお金がかかる（従量課金）場合もある。
    
- 勝手に使われないように、Keyは厳重に管理する必要がある。 **発表者ノート**: Open-Meteoは太っ腹ですが、世の中は甘くありません。会員証が必要です。
    

### スライド 17: Keyの隠し場所（.env）

**タイトル**: 鍵は玄関マットの下に置くな **内容**:

- API Keyをコードに直接書くと、GitHubで公開された時に**全世界に流出**する（数秒で不正利用される）。
    
- **`.env` ファイル** に書いて、Gitにはアップしないのが鉄則。
    
- AIに聞こう：**「API Keyを安全に管理する方法を教えて」** **発表者ノート**: 「パケ死」ならぬ「クラウド破産」を防ぐために、Keyの管理だけは慎重に。AIが正しいやり方を教えてくれます。
