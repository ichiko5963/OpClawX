### SECTION 01: 「コード」って何？（15分）

#### 1.1 コードとは何か？

**「コード（Code）」とは？**

コードは、**「コンピュータへの命令書」**です。

コンピュータは日本語がわかりません。だから、**コンピュータがわかる特別な言葉**で命令を書く必要があります。

その「特別な言葉で書かれた命令」のことを「コード」と呼びます。

例えば、人間には「青い背景の画面を出して」って言えばいいですけど、

コンピュータには：

```css
body {
  background-color: blue;
}
```

こんな風に書かないと伝わりません。

**「うわ、やっぱり難しそう...」**

って思いましたよね？

**大丈夫。これはAIが書きます。あなたは書きません。**

#### 1.2 HTML、CSS、JavaScriptの役割

Webページを作るには、3つの言語を使います：

**HTML（エイチティーエムエル）**: 骨組み
- 見出し、段落、ボタンなどの「部品」を配置する
- 例：`<h1>タイトル</h1>` = 「これは大見出しです」

**CSS（シーエスエス）**: 見た目
- 色、大きさ、レイアウトを決める
- 例：`color: red;` = 「色を赤にする」

**JavaScript（ジャバスクリプト）**: 動き
- ボタンを押したら何か起きる、などの「動作」を制御する
- 例：`button.onclick = function() { alert('Hello!'); }` = 「ボタンを押したらアラートを出す」

**全部覚える必要はありません。** AIが全部書いてくれます。

### SECTION 02: Geminiにコードを書かせる（30分）

#### 2.1 Geminiにアクセスしよう

**【実践】Geminiにログイン**

1. ブラウザで `gemini.google.com` を開く
2. Googleアカウントでログイン
3. チャット画面が出てきたら成功！

簡単ですよね。これで**世界最高クラスの頭脳にアクセスできる**ようになりました。

#### 2.2 プロンプトの書き方

**AIに上手に頼むコツは3つ：**

**1. 役割を与える**

「あなたはプロのWebデザイナーです」
「あなたは優しい先生です」

みたいに、**最初にAIに「誰として答えてほしいか」を伝える**と、回答の質が上がります。

**2. ゴールを示す**

「猫語翻訳機を作りたい」
「初心者向けに説明してほしい」

**何をしたいのか**を具体的に伝えましょう。

**3. 雰囲気（バイブス）を伝える**

「カッコいい感じで」
「可愛らしく」
「シンプルに」

**どんなノリで**作ってほしいかを伝えると、AIはそれを汲み取ってくれます。

#### 2.3 実際にコードを書かせてみよう

**【実践】Geminiにコードを書かせる**

Geminiに、こう打ち込んでみてください：

```
「猫語翻訳機」を作りたいです。

機能：
- ボタンを押すと、ランダムで猫語（ニャー、ゴロゴロ、シャーなど）が表示される
- ジョークアプリなので、適当で面白い感じで

デザイン：
- 宇宙船のコックピットみたいな、カッコいい近未来的なデザイン
- 背景は暗め、ボタンは光ってる感じ

HTMLとCSS、JavaScriptを使って、
全部を1つのファイル（index.html）にまとめて書いてください。
```

---

Geminiが「わかりました」って言って、**ズラーッと英語の文字列（コード）を出してきた**と思います。

**これが「コード」です。**

「うわ、めっちゃ長い...読めない...」

**読まなくていいです。**

読むのはあなたじゃありません。コンピュータが読みます。

### SECTION 03: コピペでWebページを作る（40分）

#### 3.1 コードをコピーする

**【手順】コードをコピーする**

1. Geminiが出してきたコードの右上に「コピー」ボタンがあるはず
2. それを押して、コードをコピーしてください

#### 3.2 メモ帳を開く

**【手順】メモ帳を開く**

- **Windowsの人**：「メモ帳」を開く
- **Macの人**：「テキストエディット」を開く（プレーンテキストモードにしてください）

※ワードやPagesはダメです。「メモ帳」「テキストエディット」みたいな、シンプルなやつを使ってください。

#### 3.3 コードを貼り付ける

**【手順】コードを貼り付ける**

さっきコピーしたコードを、メモ帳にペースト（貼り付け）してください。

- Ctrl+V（Windowsの人）
- Command+V（Macの人）

#### 3.4 名前をつけて保存する

**【手順】名前をつけて保存する**

ここが**超重要**です。

ファイル名を **`cat.html`** にして保存してください。

**保存場所は「デスクトップ」にしましょう。**

**「.html」がポイントです！**

`.txt`（テキスト）じゃなくて、**`.html`（Webページ）**にすることで、コンピュータは「これはWebページなんだな」と認識してくれます。

#### 3.5 ブラウザで開く

**【手順】ブラウザで開く**

デスクトップを見てください。

`cat.html` というファイルができていますよね？

これを**ダブルクリック**してください。

---

**どうですか？**

ブラウザ（ChromeやSafari）が開いて、**あなたが作った（というかAIが作った）Webページが表示された**はずです。

ボタンを押してみてください。「ニャー」とか出ますか？

---

**おめでとうございます！！🎉**

**あなたは今、人生で初めてWebページを作りました！**

コードは1文字も書いていません。**AIに頼んで、コピペしただけ**です。

これが「バイブコーディング」です。

### SECTION 04: 気に入らないところを直す（30分）

#### 4.1 修正を頼む

「うーん、ボタンがちょっと小さいな...」
「色をもうちょっと変えたいな...」

そう思ったら、**Geminiに文句を言えばいいんです**。

**【実践】修正を頼む**

```
ボタンが小さすぎる！
画面の半分くらいの大きさにして、もっと光らせて！
あと、背景の黒がちょっと暗すぎるから、少し紫っぽくして。
```

Geminiは「わかりました」って言って、**新しいコードを出してきます**。

それをまた、コピペして、保存し直せばOKです。

#### 4.2 修正のサイクルを体験する

**【やってみよう】**

1. 何か1つ、気に入らないところを見つけてください
2. Geminiに「ここ直して」と頼んでください
3. 新しいコードをコピペして、保存し直してください
4. ブラウザをリロード（更新）して、変わっているか確認してください

### SECTION 05: 「コピペ開発」の面倒くささを知る（10分）

#### 5.1 面倒くささを実感する

今、何回かコピペ→保存→リロードをやりましたよね。

**正直、面倒くさくなかったですか？**

「いちいちコピーして、メモ帳に貼り直して、保存して...」

**この「面倒くさい」という感覚、大事にしてください。**

実は、この「面倒くささ」を解消するために、**もっと便利な道具**があるんです。

次回から、その「便利な道具」を使い始めます。

今日「手動でコピペする」体験をしたからこそ、**その道具のありがたみがわかる**ようになります。
