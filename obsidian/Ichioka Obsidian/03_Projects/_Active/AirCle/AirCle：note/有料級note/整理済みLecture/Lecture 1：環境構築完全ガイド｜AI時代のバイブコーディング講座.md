- **エラーの原因が分からない**: 環境が整っていないと、コードが正しくても動かない
- **情報が古い**: ネット上の記事は古いバージョンの情報が多く、そのまま従うと動かない
- **用語が分からない**: 「PATH」「環境変数」「ポート番号」など、初心者には意味不明な用語が飛び交う

しかし、**2025年のAI時代では、環境構築の難易度は劇的に下がりました**。なぜなら：

- **Cursorのようなオールインワンツール**が登場し、複雑な設定が不要に
- **AIアシスタント**が設定ミスを自動検出し、修正提案してくれる
- **ブラウザベースの開発環境**（CodeSandbox、StackBlitzなど）でインストール不要で開発可能

この講座では、**2025年最新版の最短ルート**で環境を整えます。

#### 1.2 バイブコーディングとは？従来の開発との違い

「バイブコーディング」とは、**AI時代の新しい開発スタイル**です。

| 従来のプログラミング | バイブコーディング |
|---------------------|-------------------|
| コードを1行ずつ書く | 自然言語でAIに指示 |
| エラーを自力で解決 | AIが自動検出・修正 |
| ドキュメントを読み漁る | AIに質問して即解決 |
| 数ヶ月の学習期間 | 数週間で実践レベル |
| 構文暗記が必須 | 概念理解が重要 |

例えば、従来なら以下のコードを理解し、暗記する必要がありました：

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const button = document.getElementById('myButton');
  button.addEventListener('click', function() {
    alert('Hello World!');
  });
});
```

バイブコーディングでは：

```
「ボタンをクリックしたら"Hello World!"と表示する機能を追加して」
```

とCursorに指示するだけで、AIが最適なコードを生成します。

### SECTION 02: PC環境の確認とブラウザのインストール（20分）

#### 2.1 Windows vs Mac：どちらが開発に適している？

結論から言うと、**どちらでも開発は可能**です。ただし、以下の特徴があります：

**Macの利点**:
- Unix系OSなので、本番環境（Linux）に近い
- ターミナル操作がシンプル
- デザイナー・クリエイター向けツールが充実
- Web開発者の多くがMacを使用しているため、情報が豊富

**Windowsの利点**:
- ゲーミングPC等、高性能マシンが安価に手に入る
- Microsoft製品（Azure、VSCodeなど）との相性が良い
- WSL2（Windows Subsystem for Linux）でLinux環境を構築可能
- 企業のシステム開発現場ではWindowsが主流

**2025年の推奨**:
- **Mac（M1/M2/M3チップ）**: 電力効率が良く、AI処理も高速
- **Windows（WSL2環境）**: コスパ重視ならこちら

#### 2.2 Google Chromeのインストールと開発者ツール

プログラミングには**Google Chrome**を使います。理由：

- **開発者ツールが最強**: Elementsタブ（HTML構造確認）、Consoleタブ（JavaScriptデバッグ）、Networkタブ（通信確認）など
- **最新Web技術のサポート**: React、Next.jsなどのモダンフレームワークに対応
- **拡張機能が豊富**: React DevTools、Redux DevToolsなど、開発を助けるツールが充実

**インストール手順**:
1. https://www.google.com/chrome/ にアクセス
2. 「Chromeをダウンロード」をクリック
3. インストーラーを起動し、指示に従う

**開発者ツールを開く方法**:
- **Windows**: `F12` または `Ctrl + Shift + I`
- **Mac**: `Cmd + Option + I`

**開発者ツールの主要タブ**:
- **Elements**: HTML構造とCSSを確認・編集
- **Console**: JavaScriptのエラーや`console.log()`の出力を確認
- **Network**: サーバーとの通信を監視
- **Application**: Cookie、LocalStorage、Session Storageを確認

#### 2.3 最低限のPCスペックとストレージ確保

**推奨スペック**:
- **CPU**: Intel Core i5以上 / Apple M1以上
- **メモリ**: 8GB以上（16GB推奨）
- **ストレージ**: 空き容量50GB以上
- **OS**: Windows 10/11 / macOS 12以降

**なぜこのスペックが必要？**:
- **Node.js**: JavaScriptの実行環境（メモリ消費大）
- **Docker**: 仮想環境（ストレージ消費大）
- **Cursor/VSCode**: リアルタイムAI補完（CPU/GPU使用）

**ストレージ確保のコツ**:

- `node_modules`フォルダを定期的に削除（プロジェクト毎に数GB消費）
- クラウドストレージ（Google Drive、Dropbox）を活用

### SECTION 03: localhost（ローカルホスト）とは何か？（25分）

#### 3.1 localhost = あなたのPCが「サーバー」になる仕組み

プログラミング初心者が最初に戸惑うのが「localhost（ローカルホスト）」です。

**簡単に言うと**:
- **localhost = あなたのPC自身**
- 通常、Webサイトは「どこか遠くのサーバー」にアクセスしますが、localhostは**自分のPCをサーバーとして動かす**こと
- これにより、インターネットに公開せずに開発・テストができる

**例えで理解**:
- **通常のWebサイト**: 東京のサーバーにアクセスして、HTMLを取得
- **localhost**: 自分のPC内でサーバーを起動し、自分のブラウザでアクセス

**なぜlocalhostを使うのか？**:
- **開発中のコードをすぐ確認できる**: ファイルを保存 → ブラウザでリロード → 即反映
- **インターネット不要**: オフラインでも開発可能
- **セキュリティ**: 外部からアクセスされないため、安全に実験できる

#### 3.2 IPアドレス「127.0.0.1」の意味

localhostには**IPアドレス**が割り当てられています。

- **localhost = 127.0.0.1**
- これは「自分自身を指すIPアドレス」として世界中で共通

**試しに確認してみよう**:
1. ブラウザで `http://localhost` と入力
2. または `http://127.0.0.1` と入力
3. 同じページが表示される（どちらも自分のPCを指している）

**ちなみに**:
- `192.168.x.x`: 自宅のWi-Fiネットワーク内での自分のPCのアドレス
- `8.8.8.8`: Googleの公開DNSサーバーのIPアドレス

#### 3.3 ポート番号の概念（3000, 5000, 8080など）

localhostを使う際、よく見るのが「ポート番号」です。

```
http://localhost:3000
http://localhost:5000
http://localhost:8080
```

**ポート番号とは？**:
- **1つのPC内で複数のサーバーを動かすための"窓口番号"**
- 同時に複数のアプリを起動する際、それぞれ別のポートを使う

**主要なポート番号**:
| ポート | 用途 |
|--------|------|
| **3000** | React、Next.jsのデフォルトポート |
| **5000** | FlaskなどのPythonフレームワーク |
| **8080** | Apache Tomcat、代替HTTPポート |
| **80** | HTTP通信の標準ポート（管理者権限必要） |
| **443** | HTTPS通信の標準ポート（管理者権限必要） |

**よくあるエラー**:
```
Error: Port 3000 is already in use
```
→ 既に別のアプリが3000番ポートを使用中。以下で解決：
- 別のポート（例: 3001）を使う
- 既存のプロセスを終了する

**Macでポートを確認する方法**:
```bash
lsof -i :3000
```

**Windowsでポートを確認する方法**:
```bash
netstat -ano | findstr :3000
```

### SECTION 04: VSCodeのインストールと初期設定（20分）

#### 4.1 VSCode（Visual Studio Code）とは？

**VSCode**は、Microsoftが開発した**無料のコードエディタ**で、世界中の開発者が使用しています。

**VSCodeの特徴**:
- **軽量で高速**: Electronベースだが、動作が軽い
- **拡張機能が豊富**: 5万以上の拡張機能（Extensions）で機能を追加
- **多言語対応**: HTML、CSS、JavaScript、Python、Go、Rustなど、ほぼ全ての言語に対応
- **Git統合**: バージョン管理がエディタ内で完結

**なぜVSCodeを学ぶのか？**:
- 次のLecture 2で学ぶ「Cursor」は、VSCodeをベースに作られている
- VSCodeの操作を理解すれば、Cursorも直感的に使える

#### 4.2 インストール手順とおすすめ拡張機能

**インストール手順**:
1. https://code.visualstudio.com/ にアクセス
2. 「Download for Windows/Mac」をクリック
3. インストーラーを起動し、指示に従う
4. 初回起動時、言語を「日本語」に設定（Japanese Language Packをインストール）

**必須拡張機能TOP 5**:
1. **Japanese Language Pack**: 日本語化
2. **Prettier**: コード自動整形（保存時に自動でコードをキレイに）
3. **Live Server**: HTMLをlocalhost上で即座にプレビュー
4. **Auto Rename Tag**: HTMLタグの開始・終了を自動で揃える
5. **Path Intellisense**: ファイルパスを自動補完

**拡張機能のインストール方法**:
1. 左サイドバーの「Extensions」アイコンをクリック（四角いブロックのアイコン）
2. 検索バーに拡張機能名を入力
3. 「Install」をクリック

#### 4.3 初めてのHTMLファイルを作成してみよう

**手順**:
1. VSCodeを起動
2. `File` → `Open Folder` → 新しいフォルダ「my-first-website」を作成して開く
3. 左サイドバーで右クリック → `New File` → `index.html`と名付ける
4. 以下のコードを入力:

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>私の最初のWebサイト</title>
</head>
<body>
    <h1>Hello, World! 🎉</h1>
    <p>これは私の最初のHTMLファイルです。</p>
</body>
</html>
```

5. `Ctrl + S`（Windows）または`Cmd + S`（Mac）で保存
6. **Live Server拡張機能**をインストール済みの場合、右下の「Go Live」ボタンをクリック
7. ブラウザで`http://localhost:5500/index.html`が自動で開き、作成したページが表示される

**Live Serverの仕組み**:
- ファイルを保存すると、ブラウザが自動リロード
- リアルタイムでHTMLの変更を確認できる

### SECTION 05: Cursorとは？次世代AIエディタの概要（15分）

#### 5.1 VSCodeとCursorの違い

**Cursor**は、**AI機能を標準搭載したVSCodeの進化版**です。

| 項目 | VSCode | Cursor |
|------|--------|--------|
| **ベース** | オープンソース | VSCodeベース（フォーク版） |
| **AI機能** | GitHub Copilot（有料）を別途インストール | GPT-4o、Claude 3.5を標準搭載 |
| **コード生成** | 拡張機能に依存 | `Cmd+K`で即座にコード生成 |
| **AIチャット** | ChatGPT等を別途開く | エディタ内で`Cmd+L`で会話 |
| **価格** | 無料（Copilotは月$10） | 無料版あり（制限付き）、Pro版は月$20 |

**Cursorの革新的な点**:
- **コードを選択 → Cmd+K → 指示 → 即生成**
- **エラーが出たら → Cmd+L → AIに質問 → 解決策提示**
- **コード全体を理解した上でAIが提案（コンテキスト理解）**

#### 5.2 なぜCursorを使うのか？AI時代の必須ツール

従来のプログラミング学習では、以下のような時間が無駄でした：

- **構文エラーの原因を探す**: セミコロン忘れ、括弧の閉じ忘れなど
- **ドキュメントを読み漁る**: 公式ドキュメントは英語が多く、初心者には難解
- **Stack Overflowで解決策を探す**: 古い情報が多く、そのまま使えないことも

**Cursorを使うと**:
- **エラーが出た瞬間にAIが原因を教えてくれる**
- **「こういう機能を追加したい」と伝えるだけで、コードが生成される**
- **既存コードを理解した上で、最適な修正を提案**

**実例**:
- **従来**: Reactでボタンを作る方法をGoogle検索 → 記事を読む → コードをコピペ → 動かない → また検索（30分）
- **Cursor**: 「Reactでボタンを作って、クリックしたらアラートを表示」→ Cmd+K → 5秒でコード生成

#### 5.3 CursorのAI機能プレビュー（次の講座の予告）

Cursorには主に3つのAI機能があります：

1. **Cmd+K（コマンドK）**: コード生成・編集
   - 例: 「このボタンを青くして」「エラーを修正して」
   
2. **Cmd+L（コマンドL）**: AIチャット
   - 例: 「このコードの意味を教えて」「なぜエラーが出るの？」

3. **Tab補完**: コードの続きを自動予測
   - 例: `const handleClick = `と打つと、`() => { ... }`を自動補完

### SECTION 06: 実践演習：localhostでHTMLを起動する（25分）

#### 6.1 Python3を使った最もシンプルなlocalhostサーバー起動

実は、**Pythonがインストールされていれば、1コマンドでlocalhostサーバーを起動できます**。

**Windowsの場合**:
```bash
# コマンドプロンプトを開く
cd C:\Users\YourName\Desktop\my-first-website
python -m http.server 8000
```

**ブラウザで確認**:
```
http://localhost:8000
```

**起動中のサーバーを停止する方法**:
- ターミナル/コマンドプロンプトで `Ctrl + C`

#### 6.2 Live Server拡張機能を使った即座のプレビュー

VSCodeの**Live Server拡張機能**を使えば、もっと簡単です。

**手順**:
1. VSCodeでHTMLファイルを開く
2. 右下の「Go Live」ボタンをクリック
3. 自動的に`http://localhost:5500`でブラウザが開く
4. HTMLを編集して保存すると、ブラウザが自動リロード

**Live Serverの便利機能**:
- **ホットリロード**: ファイル保存時に自動更新
- **複数デバイス対応**: スマホからも`192.168.x.x:5500`でアクセス可能
- **エラー表示**: 構文エラーがあればブラウザに表示

#### 6.3 【演習】初めてのHTMLをブラウザで表示する

**課題**:

    </ul>

</html>

**確認ポイント**:
- [ ] `http://localhost:5500`または`http://localhost:8000`でページが表示される
- [ ] 名前、自己紹介、趣味リストが正しく表示される
- [ ] HTMLを編集して保存すると、ブラウザが自動更新される（Live Serverの場合）

### SECTION 07: トラブルシューティング：よくあるエラーと解決法（20分）

#### 7.1 「ポートが既に使用されています」エラー

**エラーメッセージ**:
```
Error: listen EADDRINUSE: address already in use :::3000
```

**原因**:
- 既に別のアプリが同じポート（例: 3000番）を使用している

**解決法1: 別のポートを使う**
```bash
# ポート3001を使う
python3 -m http.server 3001
```

**解決法2: 既存のプロセスを終了する**
```bash
# Macの場合
lsof -ti:3000 | xargs kill -9

# Windowsの場合
netstat -ano | findstr :3000
# 表示されたPID（プロセスID）をメモして
taskkill /PID <PID番号> /F
```

#### 7.2 「localhost refused to connect」エラー

**原因**:
- サーバーが起動していない
- ファイアウォールがlocalhostをブロックしている

**解決法**:
1. ターミナル/コマンドプロンプトでサーバーが起動中か確認
2. 以下のコマンドでlocalhostにアクセス可能か確認:
```bash
curl http://localhost:3000
```
3. ファイアウォール設定を確認（Windows Defenderなど）

#### 7.3 HTMLファイルが更新されない（キャッシュ問題）

**現象**:
- HTMLを編集して保存したのに、ブラウザで変更が反映されない

**原因**:
- ブラウザが古いHTMLをキャッシュ（一時保存）している

**解決法**:
- **スーパーリロード**で強制的にキャッシュをクリア
  - **Windows**: `Ctrl + Shift + R`
  - **Mac**: `Cmd + Shift + R`

**恒久的な対策**:
- Chromeの開発者ツールを開いた状態では、キャッシュが無効化される
- `F12` → Networkタブ → 「Disable cache」にチェック
