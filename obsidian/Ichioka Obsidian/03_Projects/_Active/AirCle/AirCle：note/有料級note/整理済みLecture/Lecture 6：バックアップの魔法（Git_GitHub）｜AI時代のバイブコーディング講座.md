## 📚 セクション構成

### SECTION 01: Git = 「セーブポイント」（20分）

#### 1.1 Gitとは何か？

**「Git（ギット）」とは？**

Gitは、**ゲームの「セーブポイント」**だと思ってください。

ファイルを上書き保存すると、過去の状態は消えます。

**Git** は、過去のあらゆる時点に「セーブポイント（コミット）」を作れます。

失敗したら、昨日のセーブポイントまで時間を巻き戻せます。

「バグった！終わった！」とならないためのタイムマシン。それがGitです。

#### 1.2 Gitの3段階

Gitには3つの段階があります：

1. **作業場（Working Tree）**: いま書いている場所。散らかっている。
2. **カゴに入れる（Staging）**: 「これを保存するぞ」と選んだもの置き場。
3. **セーブする（Commit）**: 「メッセージ」をつけて保存確定。倉庫に格納。

いきなりセーブされるわけじゃありません。「カゴに入れて（Add）」から「レジを通す（Commit）」という2段階があることを覚えておいてください。

### SECTION 02: AntigravityでGitを使う（40分）

#### 2.1 黒い画面は使わない

昔のエンジニアは `git add .` `git commit...` とコマンドを打っていました。

今は **Antigravity（VS Code）の「ソース管理」タブ** で全部できます。

左側のメニューにある「枝（ブランチ）」のようなマークをクリック。

便利な時代になりました。難しいコマンドは覚えなくていいです。ポチポチするだけで終わります。

#### 2.2 最初のコミット

**【実践】初セーブ**

1. 枝マークのタブを開く。
2. 「変更」欄にファイルがいっぱいあるはず。
3. **「＋」ボタン**（変更をステージング）を押す ➡ カゴに入った状態。
4. メッセージ欄に **"First Commit"** と書く。
5. **「コミット」ボタン** を押す。

これでリストが消えましたね？ それは「全ての変更が記録された」からです。おめでとうございます、最初のセーブポイントが作られました。

---

### SECTION 03: GitHubにアップロード（40分）

#### 3.1 GitHubとは？

**「GitHub（ギットハブ）」とは？**

- Gitは「自分のPC内」の履歴。
- **GitHub** は、それを「クラウド（ネット上）」に保存する場所。
- 全世界のエンジニア（やAI）がコードを公開・共有している。
- **デプロイ（Vercel）** も、ここからコードを引っ張っていく。

GitHubは「エンジニアのインスタグラム」です。自分の書いたコードをアップして、世界に見せたり、保存したりします。

#### 3.2 GitHubにPush

**【実践】倉庫に送る**

1. `github.com` にログイン。
2. 右上の **「+」** -> **「New repository」**。
3. Repository name: `my-vibe-app`
4. Public（公開）か Private（自分だけ）を選ぶ（練習ならPublicでOK）。
5. **「Create repository」**。
6. GitHubの画面に表示されるコマンドをコピーして、Antigravityのターミナルで実行。

これであなたのコードはクラウドに保存されました。もうPCを窓から投げ捨てても大丈夫です（投げないでください）。

#### 3.3 環境変数の罠

**【超重要】鍵は送らない**

GitHubを見てみましょう。**`.env` ファイルがありませんね？**

これは `.gitignore` という設定ファイルが「鍵は送るな！」と止めてくれているからです。

**絶対にAPIキーをGitHubに上げてはいけません。**（数秒で盗まれて悪用されます）

Antigravity賢いですね。危険物は自動で検知して、倉庫への配送をブロックしてくれています。
