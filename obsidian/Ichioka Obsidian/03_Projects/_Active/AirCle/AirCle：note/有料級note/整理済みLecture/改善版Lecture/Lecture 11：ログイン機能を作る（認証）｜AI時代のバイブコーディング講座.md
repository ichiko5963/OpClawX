## 📚 セクション構成

### SECTION 01: 認証とは何か？（25分）

#### 1.1 認証とは何か？

さて、データは保存できるようになりました。

でも、今の状態だと、世界中の全員が同じTodoリストを見ている状態（公開掲示板）です。

「自分だけのデータ」 にするために、ログイン機能をつけましょう。

#### 1.2 認証の例え

認証の例え：

- 認証なし = 公開掲示板（誰でも見れる、誰でも書ける）
- 認証あり = 自分の部屋（自分だけが見れる、自分だけが書ける）

認証機能をつければ、自分だけのデータを守れます。

#### 1.3 なぜ認証が必要なのか？

なぜ認証が必要なのか？

1. プライバシー保護：他人に見られたくないデータを守る
2. データの分離：自分だけのデータを管理できる
3. セキュリティ：他人が勝手にデータを変更できない

全部覚える必要はありません。 AIが全部やってくれます。

---

### SECTION 02: マジックリンクの設定（40分）

#### 2.1 マジックリンクとは？

「マジックリンク」とは？

マジックリンクは、パスワード不要のログイン方法です。

従来のログイン：
1. メールアドレスを入力
2. パスワードを入力
3. ログイン

マジックリンク：
1. メールアドレスを入力
2. メールに届いたリンクをクリック
3. ログイン完了

パスワードを覚える必要がありません。

#### 2.2 なぜマジックリンクを使うのか？

なぜマジックリンクを使うのか？

1. パスワード不要：忘れる心配がない
2. 実装が爆速：Supabaseの設定ボタンを1つ押すだけ
3. セキュリティ：メールを見れる本人しか入れないので、実はパスワードより安全

「一番簡単で、一番安全」。 これを使わない手はありません。

#### 2.3 マジックリンクの設定

【実践】受付カウンター設置

1. Supabaseの左メニュー `Authentication` -> `Providers`
2. `Email` を選択
3. `Enable Email provider`: ✅ ON
4. `Confirm email`: ❌ OFF（開発中はOFF推奨！）
5. `Site URL` にAntigravityのプレビューURLを設定

「メール確認」をOFFにすると、登録メールを踏まなくてもログインできます。練習のテンポが良くなります。

### SECTION 03: ログイン画面を作る（40分）

#### 3.1 ログイン画面の作成

【実践】玄関を作る

Antigravityのエージェントに指示：

```
ログイン画面（Login.jsx）を作って。
Supabaseの signInWithOtp を使って、マジックリンクを送るようにして。
送信後は「メールを確認してください」と表示して。
```

OTP（ワンタイムパスワード）というのがマジックリンクのことです。

AIに任せれば適切なコードを書いてくれます。

#### 3.2 ログイン画面の構成

ログイン画面の構成：

1. メールアドレス入力欄
2. 「ログイン」ボタン
3. 「メールを確認してください」メッセージ

全部覚える必要はありません。 AIが全部やってくれます。

#### 3.3 動作確認

【実践】動作確認

1. メールアドレスを入力
2. 「ログイン」ボタンをクリック
3. メールに届いたリンクをクリック
4. ログイン完了！

どうですか？

ログインできましたか？

これで、自分だけのデータを守れるようになりました。

### SECTION 04: セキュリティの設定（RLS）（40分）

#### 4.1 RLSとは？

ログインはできました。でも、データはまだ「みんなの広場」にあります。

RLS（Row Level Security） は、データベース側でブロックする仕組みです。

#### 4.2 RLSの例え

RLSの例え：

- RLSなし = 公開掲示板（誰でも見れる、誰でも書ける）
- RLSあり = 自分の部屋（自分だけが見れる、自分だけが書ける）

RLSを設定すれば、自分だけのデータを守れます。

#### 4.3 RLSの設定

【実践】結界を張る

AIに指示：

```
user_id が自分のIDと一致する行だけ、SELECT/INSERT/DELETE できるポリシーを作って
```

SQLが書けなくても大丈夫。 AIアシスタントに日本語で頼めば、完璧なセキュリティルールを作ってくれます。

#### 4.4 動作確認

【実践】侵入実験

1. 普通のブラウザで「Aさん」としてタスクを書く
2. シークレットモード を開く
3. 「Bさん（別のメアド）」としてログインする
4. Aさんのタスクが1つも見えなければ成功！

世界が分断されました。これで、誰に見せても恥ずかしくない安全なアプリになりました。

## 📚 今日覚えた用語

認証
  - 意味: 自分だけのデータを守る仕組み
  - 例え: 自分の部屋の鍵

マジックリンク
  - 意味: パスワード不要のログイン方法
  - 例え: メールに届いたリンクをクリックするだけ

OTP
  - 意味: ワンタイムパスワード（マジックリンクのこと）
  - 例え: 1回だけ使えるパスワード

RLS
  - 意味: データベースのセキュリティ機能
  - 例え: 自分だけのデータを守る結界

user_id
  - 意味: ユーザーのID（ログインした人の識別子）
  - 例え: 自分のIDカード

ポリシー
  - 意味: セキュリティルール
  - 例え: 「自分だけが見れる」というルール
