## 📚 セクション構成

### SECTION 01: 「ファイル」って何？（20分）

#### 1.1 ファイルとは何か？

まず、「ファイル」から。

「ファイル」とは？

ファイルは、「デジタルな1枚の紙」だと思ってください。

紙には何が書けますか？

- 文章（作文、日記、メモ）
- 絵（イラスト、写真）
- 表（成績表、家計簿）
- 音楽の楽譜

いろいろ書けますよね。

パソコンの中にも、同じように「デジタルな紙」がたくさんあります。それが「ファイル」です。

#### 1.2 ファイルには必ず「名前」がついている

ファイルには必ず「名前」がついています。

例えば：
- `旅行の思い出.jpg`（写真ファイル）
- `レポート.docx`（Word文書ファイル）
- `売上.xlsx`（Excelファイル）
- `音楽.mp3`（音楽ファイル）

ここで気づきました？ファイル名の後ろに、ピリオド（.）と英語がついてますよね？

---

### SECTION 02: 「拡張子」って何？（20分）

#### 2.1 拡張子とは何か？

この`.jpg`とか`.docx`とかの部分を、「拡張子（かくちょうし）」と言います。

「拡張子」とは？

拡張子は、「このファイルは何のファイルですか？」というラベル（名札）です。

人間で言うと、名札に「山田太郎（営業部）」って書いてあるようなもの。「この人は営業の人なんだな」ってわかりますよね。

ファイルも同じで、

- `.jpg` → 「これは写真ですよ」
- `.mp3` → 「これは音楽ですよ」
- `.docx` → 「これはWord文書ですよ」
- `.html` → 「これはWebページですよ」

この最後の`.html`がめちゃくちゃ大事です。後で使います。

#### 2.2 よく使う拡張子

プログラミングでよく使う拡張子：

.html
- 意味: Webページの骨組み
- 用途: Webページを作る

.css
- 意味: Webページの見た目
- 用途: 色やレイアウトを決める

.js
- 意味: Webページの動き
- 用途: ボタンを押したら何か起きる

.tsx
- 意味: Reactのコンポーネント
- 用途: 部品を作る

.md
- 意味: Markdown（マークダウン）
- 用途: このnoteもMarkdownで書かれている

全部覚える必要はありません。 AIが全部教えてくれます。

### SECTION 03: 「フォルダ」って何？（20分）

#### 3.1 フォルダとは何か？

次は「フォルダ」です。

「フォルダ」とは？

フォルダは、「ファイルを入れる箱（引き出し）」です。

部屋の整理を想像してください。

書類をそのまま床にバラバラに置いてたら、どこに何があるかわかりませんよね？

だから、「仕事の書類はこの引き出し」「旅行の写真はこの箱」って分けますよね。

パソコンでも同じです。

- 「写真」フォルダ → 写真ファイルを入れる
- 「仕事」フォルダ → 仕事のファイルを入れる
- 「音楽」フォルダ → 音楽ファイルを入れる

#### 3.2 フォルダの中にフォルダを入れる

フォルダの中に、さらにフォルダを入れることもできます。

「写真」フォルダの中に、「2024年」フォルダと「2025年」フォルダを作って、さらに分けたりね。

これが「階層（かいそう）」と呼ばれる構造です。

例えで理解：

- マンション（フォルダ）
  - 1階（フォルダ）
    - 101号室（ファイル）
    - 102号室（ファイル）
  - 2階（フォルダ）
    - 201号室（ファイル）

こんな感じで、入れ子構造になっています。

### SECTION 04: 「パス」って何？（30分）

#### 4.1 パスとは何か？

さて、ここからが今日の山場です。

「パス（Path）」という概念。

これがわかると、プログラミングの世界がグッと見やすくなります。

「パス」とは？

パスは、「ファイルの住所」です。

#### 4.2 住所の例え

例えば、友達の家に行くとき、住所を教えてもらいますよね。

「東京都渋谷区神宮前1-2-3 ABCマンション 301号室」

この住所があれば、世界中のどこからでもその場所にたどり着けます。

パソコンの中のファイルにも、同じように「住所」があります。

それが「パス」です。

#### 4.3 パスの具体例

例えば、こんな感じ：

```
/Users/tanaka/Documents/仕事/報告書.docx
```

これを分解すると：

- `/Users/tanaka/` → 田中さんのホームフォルダ（マンションの棟）
- `Documents/` → 「書類」フォルダ（フロア）
- `仕事/` → 「仕事」フォルダ（部屋）
- `報告書.docx` → ファイル（部屋の中にある紙）

「/（スラッシュ）」は、「の中の」という意味だと思ってください。

`/Users/tanaka/Documents/` = 「Usersの中のtanakaの中のDocumentsの中」

#### 4.4 MacとWindowsの違い

Macの場合：
```
/Users/あなたの名前/Desktop/myapp/index.html
```

Windowsの場合：
```
C:\Users\あなたの名前\Desktop\myapp\index.html
```

違いは：
- Macは `/`（スラッシュ）を使う
- Windowsは `\`（バックスラッシュ）を使う

でも、Antigravityを使えば、どちらでも同じように動きます。気にしなくてOKです。

### SECTION 05: なぜパスが大事なの？（20分）

#### 5.1 AIに指示を出すときに必要

「なんでこんな面倒なこと覚えるの？」

それはね、AIに指示を出すときに必要になるからです。

例えば、AIに「さっき作ったファイルを修正して」って言いたいとき。

「どのファイル？」って聞かれますよね。

その時に「えっと...デスクトップの...あの...」ってなると困ります。

でも、パスを知っていれば、

「`/Users/tanaka/Desktop/myapp/index.html`を修正して」

って正確に伝えられます。

AIは住所さえわかれば、一発でそのファイルを見つけて修正してくれます。

#### 5.2 実践：パスを確認してみよう

【やってみよう】

1. デスクトップに「test」というフォルダを作ってみましょう
2. その中に「memo.txt」というファイルを作ってみましょう
3. そのファイルのパスは？ → `/Users/あなたの名前/Desktop/test/memo.txt`

Macの人：Finderでファイルを右クリック→「情報を見る」→「場所」を見るとパスがわかります

Windowsの人：エクスプローラーでファイルを右クリック→「プロパティ」→「場所」を見るとパスがわかります
