## 📚 セクション構成

### SECTION 01: プログラミングの「古い常識」を捨てる（20分）

#### 1.1 古い常識とは？

まず、あなたが持っている「プログラミングのイメージ」を捨ててください。

❌ 古い常識
- 英語の呪文（「コード」って呼ばれるやつ）を何千行も暗記しないといけない
- 数学が得意じゃないと無理
- キーボードを高速でカタカタ打てないとダメ

これ、全部嘘です。というか、2023年までは本当だったんですけど、今は違います。

なぜかって？

AIが出てきたからです。

#### 1.2 AIとは何か？

ここで「AI」って言葉が出てきましたね。

「AI（エーアイ）」とは？

AIは「Artificial Intelligence」の略で、日本語だと「人工知能」です。簡単に言うと、「めちゃくちゃ賢いロボット」みたいなもの。

ChatGPTとかGeminiとか聞いたことありますか？あれがAIです。

このAIがね、プログラミングをめちゃくちゃ上手に書けるんですよ。

具体例で理解しよう：

例えば、人間には「青い背景の画面を出して」って言えばいいですけど、

コンピュータには：
```css
body {
  background-color: blue;
}
```

こんな風に書かないと伝わりません。

「うわ、やっぱり難しそう...」

って思いましたよね？

大丈夫。これはAIが書きます。あなたは書きません。

### SECTION 02: 「書く時代」から「頼む時代」へ（25分）

#### 2.1 昔のプログラマー vs 今のバイブコーダー

昔のプログラマー
```
「よし、ログイン機能を作るぞ。まずはデータベースの設計して、
認証のライブラリをインポートして、セッション管理を...」
（何時間もキーボードをカタカタ）
```

今のバイブコーダー（あなた）
```
「ねえAI、ログイン機能作って。Googleでログインできるやつ。
デザインはシンプルでモダンな感じで頼むわ」
```

これだけです。日本語で頼むだけ。

AIが「はい、わかりました」って言って、何千行ものコードを数秒で書いてくれます。

#### 2.2 なぜ「頼む時代」になったのか？

理由1: AIの性能が劇的に向上した

2023年頃から、AIがコードを書く能力が爆発的に向上しました。

- ChatGPT: OpenAIが開発したAI。コード生成が得意。
- Gemini: Googleが開発したAI。日本語がめちゃくちゃ上手。
- Claude: Anthropicが開発したAI。長いコードを理解するのが得意。

理由2: AIエディタが登場した

- Cursor: AI機能を標準搭載したエディタ
- Antigravity: Googleが作った、Geminiが最初から住んでいるエディタ
- GitHub Copilot: Microsoftが作ったAI補完ツール

これらを使えば、エディタの中でAIと会話しながら開発できます。

理由3: ノーコードツールが進化した

- v0: 言葉だけでUIデザインを作る
- Manus: 言葉だけでアプリを全自動生成
- Genspark: 検索するだけでサービスを作る

「書く」必要がなくなったんです。

### SECTION 03: 「バイブコーディング」って何？（20分）

#### 3.1 バイブ（Vibe）とは？

じゃあ、「バイブコーディング」って何なの？って話ですよね。

「バイブ（Vibe）」っていうのは、英語で「雰囲気」「ノリ」「フィーリング」みたいな意味です。

クラブで音楽に乗って踊ってる時の、あの「いい感じ〜」っていうノリのことを「Good Vibes」って言ったりします。

「バイブコーディング」は、その「ノリ」で開発すること。

つまり、「完璧を目指さず、AIとノリノリで会話しながら、いい感じにアプリを作っちゃおう」っていうスタイルです。

#### 3.2 バイブコーディングの特徴

バイブコーディングの特徴：

1. 完璧を目指さない
   - 「とりあえず動くもの」を作る
   - 後から直せばいい

2. AIと会話しながら作る
   - 「ここ直して」「もっとこうして」
   - 何度でも修正できる

3. 楽しむことが最優先
   - 難しいことは考えない
   - 「いい感じ」で進める

従来のプログラミングとの違い：

従来のプログラミング：
- コードを1行ずつ書く
- エラーを自力で解決
- ドキュメントを読み漁る
- 数ヶ月の学習期間
- 構文暗記が必須

バイブコーディング：
- 自然言語でAIに指示
- AIが自動検出・修正
- AIに質問して即解決
- 数週間で実践レベル
- 概念理解が重要

### SECTION 04: あなたの役割（20分）

#### 4.1 AIだけじゃダメな理由

「じゃあ、人間は何するの？AIが全部やるなら、僕ら要らなくない？」

いい質問です。

実は、AIだけじゃダメなんですよ。

AIは超優秀な従業員みたいなものなんですけど、「何を作るか」は自分で決められないんです。

#### 4.2 シェフの例え

例えるなら、AIは「最高の腕を持つシェフ」です。

でも、「今日何が食べたいですか？」って聞かれて、「何でもいいです」って言ったら、シェフも困りますよね。

「今日は和食の気分。あっさりしたやつで、でも少しボリュームあるやつがいいな」

こう言えば、シェフは「じゃあ、鯛の塩焼きと炊き込みご飯でどうですか？」って提案してくれる。

あなたの仕事は、「何を作りたいか伝えること」と「出てきたものがOKかNGか判断すること」です。

#### 4.3 あなたの仕事は3つだけ

まとめると、あなたの仕事は3つだけ：

1. 企画する（どんなアプリがあったら便利かな？と妄想する）
2. 指示する（AIに日本語で「こういうの作って」と頼む）
3. ジャッジする（「いいね！」か「ここ直して」か決める）

コードは書きません。

これが「バイブコーディング」です。

### SECTION 05: 必要なスキル（15分）

#### 5.1 特別なスキルは要らない

「特別なスキル要りますか？」

要りません。でも、あると助かるものは3つあります。

#### 5.2 1. 日本語力（言葉にする力）

AIに「なんか、いい感じにして」って言っても、AIも困っちゃいます。

「ボタンの色を赤にして、もうちょっと大きくして」みたいに、自分の要望を言葉にする力があると最高です。

これは練習すれば誰でもできます。

プロンプトのテンプレート（保存しておいて！）

```
あなたは〇〇（役割）です。
私は△△（自分の状況）です。
□□（目標・やりたいこと）をしたいです。
☆☆（雰囲気・条件）でお願いします。
```

例：
```
あなたはプロのWebデザイナーです。
私はプログラミング初心者です。
猫好きのための自己紹介サイトを作りたいです。
可愛らしくて、でもシンプルなデザインでお願いします。
```

#### 5.3 2. ほんの少しのパソコン知識

「ファイル」「フォルダ」って言葉がわかれば大丈夫。

これは次回（Lecture 2）でしっかり教えます。本当に小学生でもわかるように説明するので安心してください。

#### 5.4 3. 情熱（パッション）

「こういうアプリあったら便利だな〜」「これ作ってみたい！」

この「作りたい気持ち」が一番大事です。

AIは優秀ですけど、「何を作りたいか」のアイデアは持っていません。それを持っているのは、あなただけです。
