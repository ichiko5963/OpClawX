## 📚 セクション構成

### SECTION 01: APIって何？（30分）

#### 1.1 APIとは何か？

「API（エーピーアイ）」とは？

APIは、「レストランのウェイター」だと思ってください。

- 客（あなたのアプリ）: 「ハンバーグ（天気データ）ください」
- 厨房（サーバー）: 実際にデータを作っている場所。客は立ち入り禁止。
- ウェイター（API）: 客の注文を聞いて厨房に伝え、料理を持って帰ってくる人。

APIがないと、厨房に入れないのでデータがもらえない。

#### 1.2 APIの例え

APIの例え：

- レストラン = データを提供するサービス（天気予報、ニュースなど）
- ウェイター（API） = データを取り込む窓口
- 客（あなたのアプリ） = データを使う人

APIは、世界中のデータを取り込む窓口です。

#### 1.3 なぜAPIが必要なのか？

なぜAPIが必要なのか？

1. 世界中のデータを使える：天気予報、ニュース、株価など
2. 自分でデータを作る必要がない：既にあるデータを使える
3. リアルタイムで更新される：常に最新のデータが使える

全部覚える必要はありません。 AIが全部やってくれます。

---

### SECTION 02: Open-Meteo（天気API）（40分）

#### 2.1 Open-Meteoとは？

世の中のAPIは「登録（API Key）」が必要なものが多い（面倒くさい）。

Open-Meteo は、登録不要・無料で天気データをくれる神サービス。

練習にはこれが最適！

#### 2.2 ブラウザで実験

【実践】直叩きしてみる

1. ブラウザの新しいタブを開く。
2. アドレスバーに `https://api.open-meteo.com/v1/forecast?latitude=35.689&longitude=139.691&current_weather=true` と入力してEnter。
3. 画面に文字（JSON）がブワッと出たら成功！

見えましたか？ これが「生のデータ」です。東京の天気が返ってきました。

#### 2.3 JSONとは？

「JSON（ジェイソン）」とは？

JSONは、データの形式です。

例：
```json
{
  "current_weather": {
    "temperature": 15.5,
    "weathercode": 0
  }
}
```

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 03: Antigravityで実装（40分）

#### 3.1 部品の発注

【実践】Antigravityで実装

AntigravityのAIエージェントに指示：

```
Weather.jsx という新しいコンポーネントを作って。
Open-MeteoのAPIを使って、東京の現在の天気と気温を取得して表示して。
デザインはシンプルに、晴れなら太陽、雨なら傘のアイコンを出して。
```

「fetchを使って...」とか言わなくていいです。「Open-Meteoを使って」と言えば、AIはURLも使い方も全部知っています。

#### 3.2 動作確認

【実践】動作確認

1. Weather.jsxコンポーネントを表示
2. 天気と気温が表示される
3. 晴れなら太陽、雨なら傘のアイコンが表示される

どうですか？

外部のデータ（天気）が、アプリに表示されましたよね？

これがAPIの力です。

### SECTION 04: データ連携の真髄（30分）

#### 4.1 天気によって挙動を変える

天気を表示するだけなら、天気予報アプリを見ればいい。

「天気によって、Todoアプリの挙動を変える」 のが面白い。

例：雨なら「傘を持っていく」というタスクを自動追加する。

#### 4.2 実装例

【実践】天気連携機能

AIに指示：

```
もし天気が「雨（Rain）」だったら、
タスクリストの一番上に「🌂 傘を忘れない」というタスクを自動で表示して（保存はしなくていい）。
警告色（赤っぽい背景）で目立たせて。
```

どうですか？

天気によって、アプリの挙動が変わりましたよね？

これがデータ連携の真髄です。
