エンジニアが一番苦手なのがデザインです。でも2026年は、言葉さえあれば誰でもトップデザイナーになれます。

### SECTION 02: v0とは？（25分）

#### 2.1 v0とは何か？

「v0（ブイゼロ）」とは？

v0は、Vercel社が生んだ「Generative UI」ツールです。

v0の特徴：
- 画像生成AI（Midjourneyなど）の「Webサイト版」
- 「こんな画面作って」と言うと、絵ではなく「コード」で画面を作ってくれる
- 作られるのは、現代最強の技術セット（React + Tailwind CSS）

#### 2.2 v0の例え

v0の例え：

- Midjourney = 絵を描くAI（画像を生成）
- v0 = コードを書くAI（実際に動くUIを生成）

v0はただのお絵かきツールではありません。「実際に動く部品」を作ってくれます。

これが後の工程で最強の武器になります。

#### 2.3 なぜv0を使うのか？

なぜv0を使うのか？

1. センス不要：言葉だけでプロ級のデザインが作れる
2. コード生成：実際に動くReactコードを生成してくれる
3. 無料：個人利用なら無料で使える

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 03: v0でデザインを作る（40分）

#### 3.1 アカウント作成

【実践】v0の世界へ

1. `v0.dev` にアクセス
2. 「Log in」 をクリック（Vercelアカウントが必要）
3. 無料プランで始められる

Vercel（バーセル）とは？

Vercelは、Next.jsを開発している会社です。Web業界の神様みたいな存在です。

#### 3.2 最初のプロンプト

【実践】言葉を形にする

v0のチャット欄に、こう打ち込んでみてください：

```
iPhoneの計算機アプリのような、モダンでミニマルな電卓を作って。
ボタンは丸く、配色は黒とオレンジで。
```

プロンプトのコツ：

- 具体的に書く：「iPhoneの計算機アプリのような」
- 雰囲気を伝える：「モダンでミニマルな」
- 色を指定する：「配色は黒とオレンジで」

全部覚える必要はありません。 AIが全部やってくれます。

#### 3.3 「3つの提案」から選ぶ

【実践】A案、B案、C案から選ぶ

v0は一度に3種類のデザイン案を出してくれます。

あなたの仕事は 「選ぶ（Curating）」 ことです。

「B案の雰囲気が好きだな」→ Bをクリック

プロのデザイナーでも3案出すのは大変ですが、v0は10秒でやります。

あなたはクライアントとして、好きなものを選んでください。

### SECTION 04: デザインを修正する（40分）

#### 4.1 チャットで修正指示

【実践】もっとワガママに

選んだ案に対して、追加で指示を出せます。

- 「背景をダークモードにして」
- 「数字をもっと大きく、太くして」
- 「ボタンに影（シャドウ）をつけて立体的にして」

一度で完璧じゃなくていいんです。

「ここをこうして」と会話しながら理想に近づけていきます。

これがバイブコーディングの真骨頂です。

#### 4.2 部分修正（Selection Mode）

【実践】「ここだけ直して」

チャットだと伝わりにくい時は...

画面上のパーツを直接クリック！

「このボタンだけ、赤色に変えて」

「右上のあれ」とか言わなくていいんです。指差す（クリックする）だけで伝わります。

直感的に修正しましょう。

#### 4.3 履歴（Versions）の魔法

【実践】失敗しても大丈夫

「あ、さっきの方が良かったかも...」

心配無用。v0は全てのバージョンを保存しています。

時計アイコンから、いつでも過去のデザインに戻れます。

何度やり直してもOK。 破壊を恐れずに、ガンガン修正指示を出してください。

### SECTION 05: 画像から生成（Image to Code）（25分）

#### 5.1 スクショから作る

【実践】「これと同じにして！」

手書きのメモや、参考サイトのスクリーンショットをアップロードできます。

プロンプト：「この画像と同じようなデザインのログイン画面を作って」

v0が画像を解析し、コードで再現してくれます（クローン作成）。

言葉で説明するのが面倒なら、絵を見せればいいんです。

紙に書いた汚いラフ画でも、v0が綺麗に清書してくれます。

#### 5.2 バイブスの言語化

【実践】良い指示のコツ

色指定よりも 「雰囲気」 を伝えると、AIはより良いデザインを生成してくれます。

良い例：
- ⭕️ 「Appleの公式サイトみたいな清潔感」
- ⭕️ 「90年代のゲームみたいなレトロ感」
- ⭕️ 「高級ホテルのようなラグジュアリー感」

悪い例：
- ❌ 「色は#FF0000で」
- ❌ 「フォントサイズは16pxで」

AIは「形容詞」を理解しています。

あなたの頭の中にある「感じ」を、どんな言葉で伝えるか。色々試してみてください。

---

### SECTION 06: コードを見てみよう（15分）

#### 6.1 コードを見てみよう

【実践】美しさの正体

Code ボタンを押すと、英語の羅列が出てきます。

これが React（リアクト） と Tailwind CSS です。

今のWeb開発の「標準語」です。

「うわっ、文字がいっぱい...」

でも安心してください。読む必要はありません。

ただ「この文字の羅列が、あの美しい画面を作っているんだ」と知っておいてください。

#### 6.2 なぜ画像じゃなくてコードなの？

「絵」と「コード」の違い：

- Midjourneyの絵：綺麗だけど、ボタンを押せない（ただの画像）
- v0のコード：実際にボタンが押せる。文字を入力できる。

だから、そのままアプリとして使える！

これがv0の革命的な点です。 お絵かきではなく「部品製造機」なんです。

#### 6.3 コピーボタンの重要性

【実践】持ち帰る準備

作ったデザインは、v0の中に閉じ込めておいてはもったいない。

Copy することで、自分のパソコン（Antigravity）に持って帰れます。

v0
  - 意味: 言葉だけでデザインを作るツール

Generative UI
  - 意味: AIがUIを生成する技術
  - 例え: AIがUIを作る技術

  - 意味: AIへの指示文
  - 例え: 「こんな画面作って」

Curating
  - 意味: 選ぶこと
  - 例え: クライアントとして選ぶ

Refining
  - 意味: 磨き上げること
  - 例え: 会話しながら理想に近づける

  - 意味: 部品を組み合わせる技術
  - 例え: レゴブロック

  - 意味: デザインを書くためのCSSフレームワーク
  - 例え: デザインの標準語
