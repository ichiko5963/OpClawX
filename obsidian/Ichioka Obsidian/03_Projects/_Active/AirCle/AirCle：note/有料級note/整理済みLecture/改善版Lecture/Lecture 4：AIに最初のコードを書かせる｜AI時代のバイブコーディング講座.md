## 📚 セクション構成

### SECTION 01: 「コード」って何？（20分）

#### 1.1 コードとは何か？

まず、「コード」って言葉が出てきましたね。

「コード（Code）」とは？

コードは、「コンピュータへの命令書」です。

コンピュータは日本語がわかりません。だから、コンピュータがわかる特別な言葉で命令を書く必要があります。

その「特別な言葉で書かれた命令」のことを「コード」と呼びます。

#### 1.2 コードの例

例えば、人間には「青い背景の画面を出して」って言えばいいですけど、

コンピュータには：

```css
body {
  background-color: blue;
}
```

こんな風に書かないと伝わりません。

「うわ、やっぱり難しそう...」

って思いましたよね？

大丈夫。これはAIが書きます。あなたは書きません。

#### 1.3 HTML、CSS、JavaScriptの役割

Webページを作るには、3つの言語を使います：

HTML（エイチティーエムエル）: 骨組み
- 見出し、段落、ボタンなどの「部品」を配置する
- 例：`<h1>タイトル</h1>` = 「これは大見出しです」

CSS（シーエスエス）: 見た目
- 色、大きさ、レイアウトを決める
- 例：`color: red;` = 「色を赤にする」

JavaScript（ジャバスクリプト）: 動き
- ボタンを押したら何か起きる、などの「動作」を制御する
- 例：`button.onclick = function() { alert('Hello!'); }` = 「ボタンを押したらアラートを出す」

全部覚える必要はありません。 AIが全部書いてくれます。

#### 1.4 例えで理解

HTML、CSS、JavaScriptの関係：

- HTML = 家の骨組み（柱、壁、屋根）
- CSS = 家の見た目（色、デザイン、内装）
- JavaScript = 家の機能（電気、水道、エアコン）

この3つが組み合わさって、動くWebページができます。

### SECTION 02: Geminiにコードを書かせる（30分）

#### 2.1 前回の復習

前回、Geminiと「猫語翻訳機」とか「猫占い」とかのアイデアを出しましたよね。

今日は、それを実際に形にします。

#### 2.2 プロンプトの書き方（復習）

AIに上手に頼むコツは3つ（前回の復習）：

1. 役割を与える：「あなたはプロのWebデザイナーです」
2. ゴールを示す：「猫語翻訳機を作りたい」
3. 雰囲気（バイブス）を伝える：「カッコいい感じで」

#### 2.3 実際にコードを書かせてみよう

【実践】Geminiにコードを書かせる

Geminiに、こう頼んでみてください：

```
「猫語翻訳機」を作りたいです。

機能：
- ボタンを押すと、ランダムで猫語（ニャー、ゴロゴロ、シャーなど）が表示される
- ジョークアプリなので、適当で面白い感じで

デザイン：
- 宇宙船のコックピットみたいな、カッコいい近未来的なデザイン
- 背景は暗め、ボタンは光ってる感じ

HTMLとCSS、JavaScriptを使って、
全部を1つのファイル（index.html）にまとめて書いてください。
```

---

#### 2.4 返ってきたコードを確認

Geminiが「わかりました」って言って、ズラーッと英語の文字列（コード）を出してきたと思います。

これが「コード」です。

「うわ、めっちゃ長い...読めない...」

読まなくていいです。

読むのはあなたじゃありません。コンピュータが読みます。

あなたの仕事は：
1. AIに「こういうの作って」と頼む
2. 出てきたコードをコピーする
3. ファイルに貼り付けて保存する
4. ブラウザで開く

これだけです。

### SECTION 03: コピペでWebページを作る（40分）

#### 3.1 コードをコピーする

【手順】コードをコピーする

1. Geminiが出してきたコードの右上に「コピー」ボタンがあるはず
2. それを押して、コードをコピーしてください

コピーの方法：
- Windows: `Ctrl + C`
- Mac: `Command + C`

#### 3.2 メモ帳を開く

【手順】メモ帳を開く

- Windowsの人：「メモ帳」を開く
- Macの人：「テキストエディット」を開く（プレーンテキストモードにしてください）

※ワードやPagesはダメです。「メモ帳」「テキストエディット」みたいな、シンプルなやつを使ってください。

なぜ？

ワードやPagesは、見た目を整えるための情報が入ってしまうからです。

Webページを作るには、純粋なテキストが必要です。

#### 3.3 コードを貼り付ける

【手順】コードを貼り付ける

さっきコピーしたコードを、メモ帳にペースト（貼り付け）してください。

- Windows: `Ctrl + V`
- Mac: `Command + V`

#### 3.4 名前をつけて保存する

【手順】名前をつけて保存する

ここが超重要です。

ファイル名を `cat.html` にして保存してください。

保存場所は「デスクトップ」にしましょう。

「.html」がポイントです！

Lecture 2で「拡張子」の話をしましたよね。

`.txt`（テキスト）じゃなくて、`.html`（Webページ）にすることで、コンピュータは「これはWebページなんだな」と認識してくれます。

保存の方法：
1. 「ファイル」→「名前をつけて保存」
2. ファイル名を `cat.html` にする
3. 保存場所を「デスクトップ」にする
4. 「保存」をクリック

#### 3.5 ブラウザで開く

【手順】ブラウザで開く

デスクトップを見てください。

`cat.html` というファイルができていますよね？

これをダブルクリックしてください。

#### 3.6 結果を確認

どうですか？

ブラウザ（ChromeやSafari）が開いて、あなたが作った（というかAIが作った）Webページが表示されたはずです。

ボタンを押してみてください。「ニャー」とか出ますか？

おめでとうございます！！🎉

### SECTION 04: 気に入らないところを直す（30分）

#### 4.1 修正を頼む

「うーん、ボタンがちょっと小さいな...」
「色をもうちょっと変えたいな...」

そう思ったら、Geminiに文句を言えばいいんです。

#### 4.2 修正のプロンプト例

【実践】修正を頼む

Geminiに、こう頼んでみてください：

```
さっき作った「猫語翻訳機」のコードを修正してほしいです。

変更点：
- ボタンが小さすぎる！画面の半分くらいの大きさにして、もっと光らせて！
- 背景の黒がちょっと暗すぎるから、少し紫っぽくして
- 猫語の文字をもっと大きく、太くして
```

Geminiは「わかりました」って言って、新しいコードを出してきます。

#### 4.3 修正のサイクル

【やってみよう】修正のサイクルを体験する

1. 何か1つ、気に入らないところを見つけてください
2. Geminiに「ここ直して」と頼んでください
3. 新しいコードをコピペして、保存し直してください
4. ブラウザをリロード（更新）して、変わっているか確認してください

リロードの方法：
- Windows: `Ctrl + R` または `F5`
- Mac: `Command + R`

何度でも修正できます。 遠慮しないでください。

### SECTION 05: 「コピペ開発」の面倒くささを知る（10分）

#### 5.1 面倒くささを実感する

今、何回かコピペ→保存→リロードをやりましたよね。

正直、面倒くさくなかったですか？

「いちいちコピーして、メモ帳に貼り直して、保存して...」

この「面倒くさい」という感覚、大事にしてください。

#### 5.2 もっと便利な道具がある

実は、この「面倒くささ」を解消するために、もっと便利な道具があるんです。

次回から、その「便利な道具」を使い始めます。

今日「手動でコピペする」体験をしたからこそ、その道具のありがたみがわかるようになります。

その道具とは？

「Antigravity（アンチグラビティ）」という、AIが最初から住んでいるエディタです。

これを使えば：
- コピペ不要
- 保存も自動
- ブラウザも自動で開く
- AIと会話しながら開発できる

次回でこれを体験します！

## 📚 今日覚えた用語

コード
  - 意味: コンピュータへの命令書
  - 例え: コンピュータがわかる特別な言葉

HTML
  - 意味: Webページの骨組みを書く言語
  - 例え: 家の骨組み（柱、壁、屋根）

CSS
  - 意味: Webページの見た目（色・大きさ）を書く言語
  - 例え: 家の見た目（色、デザイン、内装）

JavaScript
  - 意味: Webページの動き（ボタンを押したら何か起きる）を書く言語
  - 例え: 家の機能（電気、水道、エアコン）

拡張子
  - 意味: ファイルの種類を示す名札
  - 例え: .html = Webページ、.txt = テキスト

リロード
  - 意味: ブラウザを更新して、最新の状態を表示する
  - 例え: `Ctrl + R` または `F5`
