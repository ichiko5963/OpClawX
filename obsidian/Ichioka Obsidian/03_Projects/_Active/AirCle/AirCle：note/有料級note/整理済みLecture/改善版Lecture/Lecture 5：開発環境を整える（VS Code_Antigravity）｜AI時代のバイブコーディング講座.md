## 📚 セクション構成

### SECTION 01: 「エディタ」って何？（25分）

#### 1.1 エディタとは何か？

まず、「エディタ」って言葉から説明します。

「エディタ」とは？

エディタは、「コードを書くための専用メモ帳」です。

前回使った「メモ帳」でもコードは書けますが、超不便なんですよ。

#### 1.2 メモ帳 vs エディタ

例えるなら、

- メモ帳 = ボールペンで紙に書く
- エディタ（VS Code） = ワープロで書く（しかも赤ペン先生が常に隣にいる）

くらいの差があります。

#### 1.3 VS Codeのすごいところ

VS Codeのすごいところ：

1. 色分け：コードの種類によって自動で色がつく（読みやすい）
2. エラー検知：間違いがあると赤線で教えてくれる（Wordのスペルチェックみたいな）
3. AI搭載：AIがコードを書いてくれる機能がついてる

特に3番目がヤバいんです。

AIがエディタの中に住んでいて、「ここのコード書いて」って頼むと、勝手に書いてくれるんです。

もうメモ帳にコピペする必要がなくなります。

#### 1.4 エディタの例え

エディタの例え：

- メモ帳 = 手書きのノート
- エディタ = パソコンのワープロ（しかもAI先生が常に隣にいる）

エディタを使うと：
- コードが色分けされて読みやすい
- 間違いを自動で教えてくれる
- AIがコードを書いてくれる
- 保存も自動
- ブラウザも自動で開く

メモ帳だと：
- 全部同じ色で読みにくい
- 間違いに気づかない
- 自分で全部書く必要がある
- 手動で保存する必要がある
- 手動でブラウザを開く必要がある

どっちが楽か、一目瞭然ですよね？

### SECTION 02: VS Code vs Antigravity（20分）

#### 2.1 VS Codeとは？

「VS Code（ブイエスコード）」とは？

VS Codeは、Microsoftが作った、世界で一番使われているエディタです。

特徴：
- 無料
- 軽量で高速
- 拡張機能が豊富（5万以上）
- 多言語対応（HTML、CSS、JavaScript、Python、Go、Rustなど）

でも、AI機能は標準搭載されていません。

AI機能を使うには、GitHub Copilot（有料、月$10）を別途インストールする必要があります。

#### 2.2 Antigravityとは？

「Antigravity（アンチグラビティ）」とは？

Antigravityは、Googleが作った、VS CodeにGeminiが最初から住んでいるバージョンです。

特徴：
- 無料
- Gemini AIが標準搭載（設定不要）
- VS Codeと同じ機能（中身はほぼ同じ）
- Googleツールと相性抜群

中身はほぼ同じです。でもAntigravityの方が、設定なしでいきなりAIが使えるので楽です。

#### 2.3 Cursorとの比較

ちなみに、「Cursor（カーソル）」という有名なエディタもあるんですが、あれは月3000円くらいかかります。

比較表：

VS Code
- AI機能: GitHub Copilot（別途インストール必要）
- 価格: 無料（Copilotは月$10）
- 特徴: 世界標準

Antigravity
- AI機能: Gemini（標準搭載）
- 価格: 無料
- 特徴: Google製、設定不要

Cursor
- AI機能: GPT-4o、Claude 3.5（標準搭載）
- 価格: 月$20
- 特徴: 高機能だが有料

初心者は無料のAntigravityで十分です。

### SECTION 03: Antigravityをインストールしよう（30分）

#### 3.1 インストール手順

【実践】Antigravityのインストール

1. ブラウザで `antigravity.google` にアクセス
2. 「Download for Desktop」 をクリック
3. ダウンロードしたファイルを開いて、指示に従ってインストール

※Googleアカウントでのログインが必要になるかもしれません。

#### 3.2 初回起動と設定

インストールできたら、Antigravityを開いてみてください。

初回起動時は、いくつか設定を聞かれますが、デフォルトのままでOKです。

設定項目：
- 言語：日本語（自動で選ばれる）
- テーマ：ダーク/ライト（お好みで）
- その他：デフォルトのままでOK

### SECTION 04: 画面の見方（30分）

#### 4.1 3分割構造を理解する

インストールできたら、Antigravityを開いてみてください。

「うわ、画面がいっぱいある...」

って思いますよね。大丈夫、慣れます。

画面は3つのエリアに分かれています：

① 左側（サイドバー）

ここは「フォルダの中身」を表示する場所。

パソコンのエクスプローラー（Finder）と同じです。

② 真ん中（エディタ）

ここが「コードを書く場所」。

ファイルを選ぶと、ここにその中身が表示されます。

③ 下側（ターミナル）

ここは「AIやパソコンに命令を送る場所」。

今は気にしなくてOK。後で使います。

#### 4.2 画面の例え

画面の例え：

- 左側（サイドバー） = 本棚（ファイルを選ぶ場所）
- 真ん中（エディタ） = 机（コードを書く場所）
- 下側（ターミナル） = 電話（AIやパソコンに命令を送る場所）

最初は「真ん中」だけ見ていればOKです。

左と下は、必要になったら使います。

#### 4.3 フォルダを開いてみよう

【実践】フォルダを開いてみよう

1. Antigravityのメニューから「File」→「Open Folder」
2. Lecture 2で作った「vibe-coding」フォルダを選択
3. 左側に「vibe-coding」フォルダが表示されたら成功！

フォルダが開けない場合：

- 「vibe-coding」フォルダがデスクトップにない場合は、新しく作ってください
- デスクトップに「vibe-coding」フォルダを作成
- そのフォルダを選択

---

### SECTION 05: AIエージェントを使う（40分）

#### 5.1 AIパネルを開く

さて、ここからがAntigravityの真骨頂。

エディタの中からAI（Gemini）に話しかけられます。

【実践】AIパネルを開く

1. 右側に 「Agent」 または 「AI」 みたいなパネルがあるはず
2. なければ、上部のメニューから探してみてください
3. チャット欄が出てきたら成功！

AIパネルが見つからない場合：

- 上部のメニューから「View」→「Command Palette」
- 「Agent」と検索
- 「Open Agent」を選択

#### 5.2 AIに指示を出す

【実践】AIに指示を出す

チャット欄にこう打ってみてください：

```
「index.html」というファイルを作って。
中身は「Hello World!」と表示するだけのシンプルなWebページにして。
```

---

#### 5.3 何が起きたか確認

何が起きましたか？

AIが勝手にファイルを作って、中身まで書いてくれたはずです。

左側のサイドバーを見てください。`index.html`ができていませんか？

これがすごいんです。

前回は「メモ帳を開いて、コピペして、保存して...」ってやってましたよね。

今は 「作って」って言うだけ。

AIが全部やってくれます。

#### 5.4 修正も簡単

【実践】修正を頼んでみよう

AIに、こう頼んでみてください：

```
さっき作った「index.html」を修正して。
「Hello World!」の下に、「これは私の最初のWebページです」という文章を追加して。
```

どうですか？

AIが勝手にコードを修正してくれましたよね？

コピペ不要、保存も自動、ブラウザも自動で開く。

これが「バイブコーディング」の真の力です。

---

### SECTION 06: プレビュー機能（20分）

#### 6.1 プレビュー機能とは？

【実践】作ったページを見てみよう

1. 右上に 「Preview」 ボタン（再生マークみたいなの）があるはず
2. クリックすると、画面の右半分にWebページが表示される

もうブラウザを別で開く必要もありません。

エディタの中で全部完結します。

#### 6.2 自動リロード

さらにすごいのは、自動リロード機能です。

コードを修正して保存すると、自動でブラウザが更新されるんです。

手動でリロードする必要がありません。

これが「開発環境」の力です。

## 🔗 次回への接続

次回（Lecture 6）では、バックアップの魔法（Git & GitHub）を学びます。

今日、Antigravityでコードを書きましたよね？

でも、もし間違えてコードを消しちゃったら、どうしますか？

「Ctrl + Z」で戻れるのは限界があります。ファイルを閉じたら終わりです。

Git（ギット）を使えば、過去のあらゆる時点に「セーブポイント」を作れます。

失敗したら、昨日のセーブポイントまで時間を巻き戻せます。

「バグった！終わった！」とならないためのタイムマシン。それがGitです。

なぜ次回でGitを学ぶのか？

コードを書いていると、「あ、動かなくなった！さっきの状態に戻したい！」と思うことがあります。

Gitを使えば、安心してコードを書けます。

「失敗しても大丈夫」という安心感があると、どんどん実験できるようになります。

次回でこれを学べば、失敗を恐れずにコードを書けるようになります。

それでは、また次回！
