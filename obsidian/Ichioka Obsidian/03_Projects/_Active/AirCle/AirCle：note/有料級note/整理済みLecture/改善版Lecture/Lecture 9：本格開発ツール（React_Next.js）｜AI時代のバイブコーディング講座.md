## 📚 セクション構成

### SECTION 01: Reactとは？（30分）

#### 1.1 Reactとは何か？

でも、今のWebアプリはもっと複雑です。

そこで生まれたのが React（リアクト）。

Reactは、画面を小さな「部品（コンポーネント）」に分けて、組み合わせる技術です。

#### 1.2 Reactの例え

Reactの例え：

- HTML = レゴブロックの説明書
- React = レゴブロックの部品
- コンポーネント = 1つの部品（ボタン、カードなど）

部品を組み合わせるだけでアプリができます。

#### 1.3 コンポーネントとは？

「コンポーネント」とは？

コンポーネントは、再利用可能な部品です。

例：
- 「ボタン部品」
- 「ヘッダー部品」
- 「カード部品」

これらを `Components` フォルダに入れておき、パズルのように組み合わせるのがReactです。

#### 1.4 なぜReactを使うのか？

なぜReactを使うのか？

1. 部品を再利用できる：一度作ったボタンは、何度でも使える
2. コードが読みやすい：部品ごとに分かれているので、わかりやすい
3. AIが得意：AIは、この「部品作り」がめちゃくちゃ得意

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 02: Next.jsとは？（30分）

#### 2.1 Next.jsとは何か？

Next.js（ネクストジェイエス） は、Reactを使って「Webサイト全体」を作るためのフレームワークです。

Reactが「エンジン」なら、Next.jsは「車体」です。

#### 2.2 React vs Next.js

React vs Next.js：

React
- 役割: 部品を作る
- 例え: エンジン
- 機能: コンポーネント

Next.js
- 役割: Webサイト全体を作る
- 例え: 車体
- 機能: ルーティング、サーバーサイドレンダリング、API Routes

Next.jsは、Reactの完全装備セットです。

#### 2.3 Next.jsの特徴

Next.jsの特徴：
- ルーティング（ページ遷移）が簡単
- サーバーサイドレンダリング（高速表示）
- API Routes（バックエンド機能）

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 03: コンポーネントを作る（40分）

#### 3.1 コンポーネントの基本構造

コンポーネントの基本構造：

```javascript
// Button.tsx
export default function Button() {
    return (
        <button>クリック</button>
    );
}
```

各要素の意味：

- `export default` = 「この部品を他の場所で使えるようにする」
- `function Button()` = 「Buttonという部品を作る」
- `return` = 「この部品が何を表示するか」

全部覚える必要はありません。 AIが全部やってくれます。

#### 3.2 コンポーネントを使う

【実践】コンポーネントを使う

AntigravityのAIエージェントに指示：

```

「クリック」というテキストが表示されるボタンにして。
```

どうですか？

AIが勝手にコンポーネントを作ってくれましたよね？

これがReactの力です。

#### 3.3 コンポーネントを組み合わせる

【実践】コンポーネントを組み合わせる

AntigravityのAIエージェントに指示：

部品を組み合わせるだけで、複雑なUIが作れます。

これがReactの真の力です。

### SECTION 04: State（ステート）の概念（30分）

#### 4.1 Stateとは何か？

「State（ステート）」とは？

Stateは、「画面を書き換えるスイッチ付きの箱」です。

普通の変数: 書き換えても、画面は変わらない（リロードが必要）。

State (`useState`): 書き換えた瞬間、画面が自動で再描画される魔法の変数。

#### 4.2 Stateの例

Stateの例：

- 「おみくじ結果」
- 「入力フォームの文字」
- 「タスクリスト」

これらは全部Stateです。

Stateが変わると、Reactが「おっ、画面更新しなきゃ！」と反応します。

#### 4.3 useStateの使い方

useStateの使い方：

```javascript
const [count, setCount] = useState(0);
```

各要素の意味：

- `count` = 現在の値（0）
- `setCount` = 値を変更する関数
- `useState(0)` = 初期値を0にする

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 05: Server Component vs Client Component（30分）

#### 5.1 2つの世界

Next.jsには「2つの世界」があります。

Server Component (厨房): サーバー側。DBに近い。安全。でも客の操作（クリック）はできない。

Client Component (客席): ブラウザ側。クリックや入力ができる。でもDBに直接触るのは危険。

#### 5.2 使い分けの基準

使い分けの基準：

- Q. データベースやAPIキーを使いますか？ → YES ➡ Server Component (デフォルト)
- Q. `onClick` や `useState` を使いますか？ → YES ➡ Client Component (`"use client"` をつける)

迷ったらこれを見てください。

最初は全部 `"use client"` でも動きますが、慣れてきたら使い分けるとプロっぽくなります。

#### 5.3 "use client"とは？

`"use client"` = 「これは厨房じゃなくて、客席（ブラウザ）で動かす部品ですよ」という宣言。

例：

```javascript
"use client"

export default function Button() {
    const [clicked, setClicked] = useState(false);
    
    return (
        <button onClick={() => setClicked(true)}>
            {clicked ? "クリック済み" : "クリック"}
        </button>
    );
}
```

全部覚える必要はありません。 AIが全部やってくれます。
