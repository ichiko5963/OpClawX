## 📚 セクション構成

### SECTION 01: Git = 「セーブポイント」（25分）

#### 1.1 Gitとは何か？

「Git（ギット）」とは？

Gitは、ゲームの「セーブポイント」だと思ってください。

ファイルを上書き保存すると、過去の状態は消えます。

Git は、過去のあらゆる時点に「セーブポイント（コミット）」を作れます。

失敗したら、昨日のセーブポイントまで時間を巻き戻せます。

「バグった！終わった！」とならないためのタイムマシン。それがGitです。

#### 1.2 ゲームのセーブポイントの例え

ゲームのセーブポイントの例え：

- 普通の保存 = 上書き保存（過去の状態は消える）
- Git = セーブポイント（過去の状態も全部残る）

例：
- 朝9時：セーブポイント1「最初の状態」
- 昼12時：セーブポイント2「ログイン機能を追加」
- 夕方6時：セーブポイント3「デザインを変更」
- 夜9時：バグ発生！「動かなくなった！」

Gitを使えば：
- セーブポイント3に戻れる
- セーブポイント2に戻れる
- セーブポイント1に戻れる

「バグった！終わった！」とならないためのタイムマシン。それがGitです。

#### 1.3 Gitの3段階

Gitには3つの段階があります：

1. 作業場（Working Tree）: いま書いている場所。散らかっている。
2. カゴに入れる（Staging）: 「これを保存するぞ」と選んだもの置き場。
3. セーブする（Commit）: 「メッセージ」をつけて保存確定。倉庫に格納。

いきなりセーブされるわけじゃありません。「カゴに入れて（Add）」から「レジを通す（Commit）」という2段階があることを覚えておいてください。

#### 1.4 3段階の例え

3段階の例え：

- 作業場（Working Tree） = 机の上（散らかっている）
- カゴに入れる（Staging） = 買い物カゴ（保存するものを選ぶ）
- セーブする（Commit） = レジで精算（保存確定）

なぜ2段階なのか？

全部を保存する必要はないからです。

例えば：
- ファイルA：完成した（保存したい）
- ファイルB：まだ作業中（保存したくない）
- ファイルC：テスト用（保存したくない）

カゴに入れる（Staging）で、保存したいものだけ選べます。

### SECTION 02: AntigravityでGitを使う（40分）

#### 2.1 黒い画面は使わない

昔のエンジニアは `git add .` `git commit...` とコマンドを打っていました。

今は Antigravity（VS Code）の「ソース管理」タブ で全部できます。

左側のメニューにある「枝（ブランチ）」のようなマークをクリック。

便利な時代になりました。難しいコマンドは覚えなくていいです。ポチポチするだけで終わります。

#### 2.2 ソース管理タブを開く

【実践】ソース管理タブを開く

1. Antigravityの左側のメニューを見てください
2. 「ソース管理」というタブ（枝のようなマーク）をクリック
3. 「変更」欄にファイルがいっぱいあるはず

「ソース管理」タブが見つからない場合：

- 左側のメニューを確認
- 「Source Control」という英語表記の場合もあります
- 枝のようなマークを探してください

#### 2.3 最初のコミット

【実践】初セーブ

1. 「変更」欄にファイルがいっぱいあるはず
2. 「＋」ボタン（変更をステージング）を押す ➡ カゴに入った状態
3. メッセージ欄に "First Commit" と書く
4. 「コミット」ボタン を押す

これでリストが消えましたね？ それは「全ての変更が記録された」からです。おめでとうございます、最初のセーブポイントが作られました。

#### 2.4 コミットメッセージの書き方

コミットメッセージの書き方：

コミットメッセージは、「何をしたか」を簡潔に書きます。

良い例：
- "First Commit"（最初のコミット）
- "ログイン機能を追加"
- "デザインを変更"
- "バグを修正"

悪い例：
- "あああ"（何をしたかわからない）
- "修正"（何を修正したかわからない）
- "更新"（何を更新したかわからない）

簡潔に、何をしたかを書けばOKです。

### SECTION 03: GitHubにアップロード（40分）

#### 3.1 GitHubとは？

「GitHub（ギットハブ）」とは？

- Gitは「自分のPC内」の履歴。
- GitHub は、それを「クラウド（ネット上）」に保存する場所。
- 全世界のエンジニア（やAI）がコードを公開・共有している。
- デプロイ（Vercel） も、ここからコードを引っ張っていく。

GitHubは「エンジニアのインスタグラム」です。自分の書いたコードをアップして、世界に見せたり、保存したりします。

#### 3.2 GitHubの例え

GitHubの例え：

- Git = 自分のPC内のセーブポイント（ローカル）
- GitHub = クラウド上のセーブポイント（ネット上）

なぜGitHubが必要なのか？

1. バックアップ：PCが壊れても、コードは残る
2. 共有：友達にコードを見せられる
3. デプロイ：Vercelなどのサービスが、ここからコードを引っ張る

GitHubは「コードの図書館」です。

#### 3.3 GitHubにPush

【実践】倉庫に送る

1. `github.com` にログイン。
2. 右上の 「+」 -> 「New repository」。
3. Repository name: `my-vibe-app`
4. Public（公開）か Private（自分だけ）を選ぶ（練習ならPublicでOK）。
5. 「Create repository」。
6. GitHubの画面に表示されるコマンドをコピーして、Antigravityのターミナルで実行。

これであなたのコードはクラウドに保存されました。もうPCを窓から投げ捨てても大丈夫です（投げないでください）。

#### 3.4 ターミナルでPushする

【実践】ターミナルでPushする

1. Antigravityの下側のターミナルを開く
2. GitHubの画面に表示されるコマンドをコピー
3. ターミナルに貼り付けて実行

コマンドの例：
```bash
git remote add origin https://github.com/あなたのユーザー名/my-vibe-app.git
git branch -M main
git push -u origin main
```

「うわ、コマンドが怖い...」

大丈夫です。コピペするだけです。

コマンドの意味（理解しなくてもOK）：
- `git remote add origin` = GitHubの場所を登録
- `git branch -M main` = ブランチ名を「main」に変更
- `git push -u origin main` = GitHubにアップロード

全部コピペするだけでOKです。

### SECTION 04: 環境変数の罠（20分）

#### 4.1 環境変数とは？

「環境変数」とは？

環境変数は、「秘密の鍵」だと思ってください。

例えば：
- SupabaseのAPIキー
- Gemini APIのキー
- その他の秘密情報

これらは絶対に他人に見せてはいけません。

#### 4.2 なぜGitHubに上げてはいけないのか？

絶対にAPIキーをGitHubに上げてはいけません。

なぜ？

- 数秒で盗まれて悪用されます
- あなたのアカウントが使われて、お金がかかる
- 他の人のデータにアクセスされる可能性がある

GitHubは公開されているので、誰でも見れます。

#### 4.3 .gitignoreとは？

GitHubを見てみましょう。`.env` ファイルがありませんね？

これは `.gitignore` という設定ファイルが「鍵は送るな！」と止めてくれているからです。

「.gitignore」とは？

`.gitignore`は、「GitHubに上げたくないファイルを指定する設定ファイル」です。

例：
```
.env
node_modules/
*.log
```

これで、`.env`ファイルはGitHubに上がりません。

Antigravity賢いですね。危険物は自動で検知して、倉庫への配送をブロックしてくれています。

#### 4.4 環境変数の管理方法

環境変数の管理方法：

1. `.env`ファイルに保存（ローカル）
2. `.gitignore`に追加（GitHubに上げない）
3. Vercelなどの本番環境に直接設定（後で学びます）

今は、`.env`ファイルを`.gitignore`に追加するだけでOKです。

## 📚 今日覚えた用語

Git
  - 意味: コードのセーブポイントを作るツール
  - 例え: ゲームのセーブポイント

GitHub
  - 意味: コードをクラウドに保存する場所
  - 例え: コードの図書館

コミット
  - 意味: セーブポイントを作ること
  - 例え: レジで精算

Push
  - 意味: GitHubにコードをアップロードすること
  - 例え: 倉庫に送る

.gitignore
  - 意味: GitHubに上げたくないファイルを指定する設定
  - 例え: 「鍵は送るな！」の設定

環境変数
  - 意味: 秘密の鍵（APIキーなど）
  - 例え: 銀行の暗証番号

Staging
  - 意味: 保存するものを選ぶこと
  - 例え: 買い物カゴに入れる
