## 📚 セクション構成

### SECTION 01: Gemini APIとは？（30分）

#### 1.1 Gemini APIとは何か？

天気APIは「事実を返すだけ」。

Gemini API は「考えて、生成する」。

違い：決まった答えがない。文脈を理解する。

#### 1.2 天気API vs Gemini API

天気API vs Gemini API：

- 天気API: ---------
- Gemini API: ------------
返すもの

- 天気API: 事実（天気、気温）
- Gemini API: 考えた結果（アドバイス、生成）
答え

- 天気API: 決まっている
- Gemini API: 決まっていない（文脈によって変わる）
例え

- 天気API: 辞書
- Gemini API: 相談相手

Gemini APIは、アプリに知能を宿します。

#### 1.3 なぜGemini APIを使うのか？

なぜGemini APIを使うのか？

1. 考えてくれる：タスクリストを見て、アドバイスをくれる
2. 生成してくれる：タスクを分解してくれる
3. 励ましてくれる：モチベーションを上げてくれる

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 02: Google AI Studio（40分）

#### 2.1 脳みその調達場所

【実践】Google AI Studio

1. `aistudio.google.com` にアクセス（Googleアカウントでログイン）。
2. 「Get API key」 をクリック。
3. 「Create API key」 でキーを発行し、コピーする。

これが「知能へのアクセス権」です。絶対に他人に教えてはいけません。

#### 2.2 APIキーとは？

「APIキー」とは？

APIキーは、「AIへのアクセス権」です。

例え：
- APIキー = 銀行の暗証番号
- Gemini API = 銀行のATM

APIキーがないと、AIを使えません。

#### 2.3 APIキーの管理

【超重要】APIキーの管理

1. `.env`ファイルに保存（ローカル）
2. `.gitignore`に追加（GitHubに上げない）
3. Vercelなどの本番環境に直接設定（後で学びます）

絶対にGitHubに上げてはいけません。

Lecture 6で学んだ`.gitignore`で、自動的にブロックされています。

### SECTION 03: AI機能の実装（50分）

#### 3.1 相談窓口を作る

【実践】AI機能の実装

AntigravityのAIエージェントに指示：

```
タスクリストの下に「🔥 修造に相談する」というボタンを作って。
押すと、現在のタスク一覧を読み取って、Gemini API（gemini-1.5-flash）を呼び出して。
プロンプトは「あなたは熱血コーチです。以下のタスクリストを見て、ユーザーを全力で励まし、具体的なアドバイスを1つください」として。
タスクリストの中身もプロンプトに含めてね。
返ってきた言葉を、吹き出しのようなデザインで画面に表示して。
```

モデルは「Flash」を使うよう指示してください。軽くて速くて無料枠が広いからです。

#### 3.2 動作確認

【実践】動作確認

1. タスクをいくつか追加
2. 「🔥 修造に相談する」ボタンをクリック
3. AIがアドバイスを返してくる

どうですか？

AIがアドバイスを返してきましたか？

これがGemini APIの力です。

#### 3.3 プロンプトエンジニアリング

「プロンプトエンジニアリング」とは？

プロンプトエンジニアリングは、「AIに上手に指示を出す技術」です。

良いプロンプトの例：

```
あなたは熱血コーチです。
以下のタスクリストを見て、ユーザーを全力で励まし、具体的なアドバイスを1つください。

タスクリスト：
- タスク1
- タスク2
- タスク3
```

悪いプロンプトの例：

```
アドバイスして
```

具体的に、役割を与えて、ゴールを示すと、AIはより良い答えを返してくれます。

### SECTION 04: JSONモード（発展）（30分）

#### 4.1 JSONモードとは？

励ましも嬉しいけど、「タスクを分解」 してくれたらもっと便利。

例：「カレーを作る」→ 「肉を買う」「野菜を切る」「煮込む」に分解して、自動でリストに追加してほしい。

#### 4.2 JSONモードの実装

【実践】JSONモード

プロンプトの最後に：

```
回答は必ずJSON形式にしてください。
例：[{"task": "野菜を切る"}, {"task": "肉を買う"}, {"task": "煮込む"}]
```

AIの答えを「読む」だけでなく「機能として使う」ための魔法です。

#### 4.3 JSONモードの例え

JSONモードの例え：

- 通常の回答 = 文章（読むだけ）
- JSONモード = データ（機能として使える）

JSONモードを使えば、AIの答えを直接アプリの機能として使えます。
