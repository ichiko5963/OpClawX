## 📚 セクション構成

### SECTION 01: データの3段階（30分）

#### 1.1 データを保存する方法には3つのレベルがある

データを保存する方法には、3つのレベルがあります。

1. State（変数）: 手のひら（リロードで消える）
2. LocalStorage: ポケット（リロードで消えないが、端末が変わると無い）
3. Cloud DB: 銀行（ネット上にある。どこからでも見れる）

今日は、「銀行（Cloud DB）」を使います。

#### 1.2 3段階の例え

3段階の例え：

- State（変数） = 手のひら（リロードで消える）
  - 例：おみくじの結果
  - リロードしたら消える

- LocalStorage = ポケット（リロードで消えないが、端末が変わると無い）
  - 例：ブラウザに保存された設定
  - PCで保存しても、スマホでは見れない

- Cloud DB = 銀行（ネット上にある。どこからでも見れる）
  - 例：Supabaseに保存されたデータ
  - PCで保存しても、スマホでも見れる

#### 1.3 なぜCloud DBを使うのか？

なぜCloud DBを使うのか？

1. どこからでもアクセスできる：PCで保存しても、スマホでも見れる
2. 複数人で共有できる：友達とデータを共有できる
3. バックアップが自動：データが消えない

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 02: Supabaseとは？（30分）

#### 2.1 Supabaseとは何か？

「Supabase（スパベース）」とは？

Supabaseは、「ネット上にExcelファイルがある」と思えばいいです。

- Table（テーブル）: Excelの「シート」
- Column（カラム）: 列（タスク名、完了フラグなど）
- アプリから、このExcelに書き込みに行くだけ

「DB構築」とか言うと怖いですが、「Excelの表を作る」だけです。

#### 2.2 Supabaseの例え

Supabaseの例え：

- Supabase = ネット上にあるExcelファイル
- Table（テーブル） = Excelのシート
- Column（カラム） = 列（タスク名、完了フラグなど）
- Row（行） = 1つのデータ（タスク1つ分）

全部覚える必要はありません。 AIが全部やってくれます。

#### 2.3 なぜSupabaseを使うのか？

なぜSupabaseを使うのか？

1. 無料枠が広い：個人開発なら無料で使える
2. AIとの相性が抜群：AIがSQLを書くのが得意
3. 認証機能もセット：ログイン機能も簡単に作れる

全部覚える必要はありません。 AIが全部やってくれます。

### SECTION 03: Supabaseの準備（40分）

#### 3.1 銀行口座を作る

【実践】Supabaseの準備

1. `supabase.com` にアクセス → GitHubアカウントでログイン
2. 「New Project」 をクリック
3. 名前：`my-todo-app`、パスワードを設定
4. 「Create Project」（数分でデータベースが爆誕）

これであなたは、世界中にサーバーを持つデータベースの管理者になりました。

#### 3.2 テーブル作成（AIに頼む）

【実践】Excelの表を作る

1. Supabaseには 「AI SQL Editor」 がある
2. チャット欄に指示： 「Todoアプリ用のテーブルを作って。タスク名(text)と、完了フラグ(boolean)と、作成日時(timestamp)が必要。」
3. 実行ボタンを押すと、テーブルができる

SQLという言語が必要ですが、AIが書くので覚えなくていいです。

#### 3.3 テーブルの例え

テーブルの例え：

- Table（テーブル） = Excelのシート
- Column（カラム） = 列（タスク名、完了フラグなど）
- Row（行） = 1つのデータ（タスク1つ分）

### SECTION 04: Antigravityと接続（40分）

#### 4.1 合言葉をコピーせよ

【実践】Antigravityと接続

AntigravityからSupabaseにアクセスするには「鍵」が必要です。

1. Supabaseの `Project Settings` -> `API` を開く
2. URL と anon key の2つをコピーする
3. Antigravityの `.env` ファイルに設定する

これが「銀行の暗証番号」みたいなものです。

#### 4.2 .envファイルとは？

「.envファイル」とは？

`.env`ファイルは、「秘密の鍵を保存するファイル」です。

例：
```
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

このファイルは絶対にGitHubに上げてはいけません。

Lecture 6で学んだ`.gitignore`で、自動的にブロックされています。

#### 4.3 接続の確認

【実践】接続の確認

AntigravityのAIエージェントに指示：

```
Supabaseに接続できているか確認して。
接続できていたら、「接続成功！」と表示して。
```

どうですか？

接続できていれば、「接続成功！」と表示されるはずです。

### SECTION 05: データの保存・読み込み（40分）

#### 5.1 銀行に書き込む

【実践】データを保存する

AntigravityのAIエージェントに指示：

```
LocalStorageはやめる。
Supabaseを使って、データの保存・読み込みをするようにコードを書き換えて。
```

エージェントが `supabase-js` をインストールし、コードを総入れ替えします。

ここがハイライトです。保存先を「ポケット」から「銀行」に切り替えます。

#### 5.2 動作確認

【実践】動作確認

1. PCでタスクを追加
2. スマホで確認
3. リアルタイムで同期されている！

どうですか？

PCで保存したデータが、スマホでも見れますよね？

これがCloud DBの力です。

#### 5.3 データの保存・読み込みの例え

データの保存・読み込みの例え：

- 保存 = 銀行に預金する
- 読み込み = 銀行から引き出す
- 更新 = 預金を変更する
- 削除 = 預金を解約する

全部覚える必要はありません。 AIが全部やってくれます。
