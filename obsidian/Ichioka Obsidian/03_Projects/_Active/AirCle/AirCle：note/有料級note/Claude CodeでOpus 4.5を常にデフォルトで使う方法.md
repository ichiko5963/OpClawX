## 第1章：Claude CodeでOpus 4.5を使う仕組み（最初に理解すべきこと）

### Claude Codeとは何か？

Claude Codeは、いわゆる「アプリ」ではありません。

**ターミナル（シェル）上で動くCLIエージェント**です。

つまり、

- **Claude Codeは**  
  👉 起動されたターミナルの「環境変数」を読む

- **ChatGPT / Codex / Geminiは**  
  👉 一切関係ない

という構造になっています。

ここを理解していないと、

「Claude Code開いた時だけ反映されるの？」

という疑問が必ず出ますが、  
答えは **「その通り」かつ「それで正解」** です。

### なぜ環境変数で設定するのか？

Claude Codeは、起動時に環境変数を読み込んで、使用するモデルを決定します。

環境変数で設定することで：

- 毎回モデルを指定する必要がない
- ターミナルを開くたびに自動的にOpus 4.5が使われる
- 他のツール（ChatGPT/Codex/Gemini）には影響しない

という利点があります。

## 第2章：Opus 4.5をデフォルト指定する環境変数

### ステップ1：環境変数を設定する

Claude Codeでは、モデルは **環境変数で指定できます**。

使うのはこれです。

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-5@20251101"
```

最初、私はこれをそのままターミナルで実行しました。

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-5@20251101"
```

すると、

「設定できました。ただしこの設定は現在のシェルセッションのみ有効です」

というメッセージが出ます。

ここで一瞬混乱しますが、  
これは **エラーではありません。完全に正常** です。

### 罠①：このままだとターミナルを閉じると消える

この `export` は、

- 今開いているターミナル
- 今のセッション

**だけ** に効きます。

つまり、

- ターミナルを閉じる
- 新しいターミナルを開く

👉 **設定は消える**

ここで「失敗した？」と思いがちですが、違います。

## 第3章：.zshrc に書いて"永久化"する

### ステップ2：.zshrc に追加する

Mac（zsh）の場合、  
環境変数を永続化する正解の場所は **.zshrc** です。

最初にやったのはこれでした。

```bash
nano ~/.zshrc
```

そして一番下に、

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-5@20251101"
```

を追加。

### 罠②：Permission denied（ここで詰まる人が多い）

Enterで保存しようとすると、こう出ました。

```
Cannot open file for writing: Permission denied
```

これは、

**「.zshrc の所有者・権限の問題」**

miniconda や過去のセットアップで、  
.zshrc が管理者権限扱いになっていると起きます。

ここで詰まって諦める人がかなり多い。

### 正解ルート：sudoで編集する

この場合の最短・安全ルートはこれです。

```bash
sudo nano ~/.zshrc
```

1. Macのログインパスワードを入力
2. 同じ行を追加
3. 保存して終了

```bash
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-opus-4-5@20251101"
```

### ステップ3：設定を即時反映

保存しただけでは、  
今開いているターミナルにはまだ反映されません。

ここで必ずこれを実行します。

```bash
source ~/.zshrc
```

### 最終確認（ここで安心していい）

```bash
echo $ANTHROPIC_DEFAULT_OPUS_MODEL
```

これが表示されれば成功です。

```
claude-opus-4-5@20251101
```

## 第6章：トラブルシューティング

### 問題1：環境変数が反映されない

**症状**：`echo $ANTHROPIC_DEFAULT_OPUS_MODEL` で何も表示されない

**解決方法**：
1. `.zshrc` に正しく追加されているか確認
2. `source ~/.zshrc` を実行
3. 新しいターミナルを開いて確認

### 問題2：Permission deniedが続く

**症状**：`sudo nano ~/.zshrc` でもPermission deniedが出る

**解決方法**：
1. `.zshrc` の所有者を確認：`ls -la ~/.zshrc`
2. 所有者を変更：`sudo chown $USER ~/.zshrc`
3. 再度 `nano ~/.zshrc` で編集

### 問題3：Claude CodeがOpus 4.5を使わない

**症状**：設定したのに別のモデルが使われる

**解決方法**：
1. 環境変数が正しく設定されているか確認：`echo $ANTHROPIC_DEFAULT_OPUS_MODEL`
2. Claude Codeを再起動
3. モデル名が正しいか確認（`claude-opus-4-5@20251101`）
