## データフロー

```
1. 収集 (n8n)
   Gmail/Calendar/Drive/Slack → API取得 → raw/ に一時保存

2. 正規化 (n8n)
   raw/ → 共通スキーマ変換 → normalized/events_YYYY-MM-DD.jsonl

3. 整流 (n8n + OpenClaw)
   normalized/ → プロジェクト分類 → daily/YYYY-MM-DD.md

4. 確認 (OpenClaw → User)
   daily digest → Telegram 通知 → ユーザー確認

5. 実行 (n8n)
   ActionQueue/pending/ → 承認済みのみ → API実行 → processed/
```

## 障害時の復旧

| 障害 | 影響 | 復旧方法 |
|------|------|----------|
| n8n 停止 | 収集停止 | Docker 再起動、cursors.json から再開 |
| API 認証失効 | 収集停止 | n8n で再認証 |
| iCloud 同期エラー | Vault 書き込み失敗 | Git から復旧、iCloud 再同期 |
| OpenClaw 停止 | 通知停止 | 収集は継続、OpenClaw 再起動後に Digest を読む |
