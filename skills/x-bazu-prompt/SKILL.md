# Xバズ投稿生成スキル v2.0（バズ分析対応版）

## 目的
Xの元投稿を分析し、過去のバズった投稿データを学習して、よりバズりやすい投稿案を生成する。

## 入力情報

### 1. 元のX投稿内容（必須）
- リライト対象の投稿テキスト
- またはX投稿URL（内容を抽出）

### 2. バズ分析シート（必須）
以下の形式で過去のバズった投稿データを提供：

```
【バズった投稿分析シート】

投稿1:
- フック: 【結論から言います】
- 文字数: 180
- いいね数: 5,347
- 構造: フック→箇条書き→つまり→未来予測
- 特徴: 無料配布系、学生向け

投稿2:
- フック: 【速報】
- 文字数: 220
- いいね数: 3,871
- 構造: 速報→結論→箇条書き→行動誘導
- 特徴: 最新ニュース、先行性

投稿3:
- フック: 正直
- 文字数: 150
- いいね数: 2,940
- 構造: 正直→問題提起→解決策→感想
- 特徴: 親しみやすいトーン

【傾向まとめ】
- 最も効果的なフック: 【結論から言います】
- 最適文字数: 150-250文字
- 必須要素: 箇条書き、つまりで解説
- トーン: 親しみやすく、断定しすぎない
- 絵文字: 🔥👇を適度に使用
```

## 出力形式

### 分析フェーズ
まず、バズ分析シートから学習：
1. 最も効果的なフックパターン
2. 最適な文字数と構造
3. 読者の反応が良かった要素
4. トーンと表現の特徴

### 生成フェーズ
分析結果を元に、元投稿を以下の5パターンでリライト：

1. **分析で最も効果的だったフック型**
   - バズ分析でNo.1だったフックを使用
   
2. **速報型（時事性重視）**
   - 【速報】など緊急性のあるフック

3. **正直型（親しみやすさ重視）**
   - 「正直」から始める親近感型

4. **保存版型（実用性重視）**
   - 【保存版】で長期的価値を強調

5. **海外トレンド型（権威性重視）**
   - 【海外で話題】で信頼感を演出

## 専門用語の噛み砕き（必須）

| 避ける表現 | 噛み砕いた表現 |
|-----------|--------------|
| 可観測性 | AIの動きが見える・記録される |
| オブザーバビリティ | 内部が見える・動きが追える |
| ネイティブ | 中身で動く・組み込みの |
| プロキシ | 中継・外から見る |
| デリゲーション | 仕事任せ・依頼 |
| コンテキスト | 文脈・状況 |
| スキーマ | 設計図・構造 |
| I/O | 入力と出力 |
| レイテンシ | 遅延・時間差 |
| パーシステント | 継続する・途切れない |
| オプティマイゼーション | 最適化・改善 |
| インタラクティブ | 対話的・双方向の |

## 投稿構造（必須）

① **フック**（冒頭3秒で心を掴む）
② **何が起きたか**（超簡潔に）
③ **解説**（箇条書きで分かりやすく）
④ **何がヤバいのか**（「つまり」で解説）
⑤ **具体例**（イメージしやすい例）
⑥ **未来の示唆**（先見性を演出）
⑦ **行動誘導**（レビュー予定など）

## フックパターン（優先順位）

バズ分析シートの結果に基づいて選択：
1. 【結論から言います】（最強）
2. 【速報】
3. 【海外で話題】
4. 【衝撃】
5. 【保存版】
6. 【実は】
7. 【知らないと損】
8. 正直

## 読みやすさのルール

- **1行15〜25文字**
- **改行多め**
- **句読点少なめ**
- **スマホ可読性重視**

悪い例：
```
OpenClawというAIエージェントは最近海外で話題になっていて様々な使い方が可能です
```

良い例：
```
OpenClawというAIエージェントが
海外でかなり話題になっています。
理由はシンプル。
「AIが自律で働く」から。
```

## 必須要素

### 1. 情報密度
以下3つを必ず含める：
- 何が起きたのか
- 何ができるのか
- 何が変わるのか

### 2. 箇条書き
途中で必ず箇条書きを入れる：
```
できること👇
・Web自動リサーチ
・競合分析
・ニュース収集
```

### 3. ヤバさの言語化
「これ何がヤバいの？」を説明：
```
つまり何が起きるかというと…
AIが
「調べるツール」
↓
「作るツール」
↓
「働くツール」
になり始めています。
```

### 4. 未来予測
必ず最後に未来視点：
```
これ
本当に数年以内に
「AIエージェントが当たり前」
になる可能性あります。
```

### 5. 拡散トリガー
保存・引用・コメントを誘発：
```
正直これ
かなり気になってるので
実際に触ってみて
レビューしようと思います。
```

### 6. 信頼感
企業名・技術名・数値を入れる：
```
・GitHubで1300スター
・72%トークン削減
・Comet開発
```

## 絵文字ルール

### 使用OK
- 🔥（強調）
- 👇（下続く感）
- 😳（驚き）
- ⚡️（スピード）

### 使用NG（情報的すぎる）
- 📱（デバイス依存）
- 📅（日付依存）
- 🔗（リンク）
- 🚨（緊急すぎる）

## 投稿の長さ

- **通常：120〜300文字**
- **スレッド/まとめ：400〜800文字可**

バズ分析シートで最もいいねが多かった文字数を基準にする。

## トーン

- **フラット**
- **断定しすぎない**
- **でもワクワク感**

バズ分析シートの成功パターンのトーンを継承する。

## 最後の一文

よく使うパターン：
- 「実際に触ってみてレビューします」
- 「使ってみて報告します」
- 「詳細解説します」

バズ分析シートで効果が高かった表現を優先。

## 実行フロー

```
1. ユーザーがX投稿内容を送信
   ↓
2. バズ分析シートを受け取る
   ↓
3. 分析シートから学習ポイントを抽出
   ↓
4. 元投稿を5パターンでリライト
   ↓
5. HTMLページ生成
   ↓
6. Vercelデプロイ
   ↓
7. URLで共有
```

## プロンプト実行時の注意

1. **分析シートが提供されない場合**
   - 標準的なAirCleパターンを使用
   - 【結論から言います】を最優先

2. **分析シートが提供された場合**
   - 必ずその傾向を優先
   - 「分析によると〜」と明示

3. **元投稿の専門用語**
   - 必ず噛み砕き表現に置き換え
   - 難しい言葉は括弧で補足

4. **バズ要素の強調**
   - 分析シートで高評価だった要素を
   - 各投稿に少なくとも1つ含める

## 出力例

### 入力
- 元投稿: 「OpenClawに可観測性プラグインが搭載されました」
- 分析シート: 【結論から言います】型が5,347いいね、箇条書き必須

### 出力
1. 【結論から言います】型（分析No.1パターン）
2. 【速報】型
3. 正直型
4. 【保存版】型
5. 【海外で話題】型

各投稿とも分析結果を反映した構成・トーン・表現を使用。
