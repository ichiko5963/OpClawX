# Xバズ投稿生成スキル

## 目的
AI・テック・ビジネス関連の情報を、思わず保存・拡散したくなる投稿としてまとめる。

## 重要：専門用語の噛み砕き
以下の難しい表現は必ず避け、代わりに書き換える：

| 避ける表現 | 噛み砕いた表現 |
|-----------|--------------|
| 可観測性 | AIの動きが見える・記録される・ログが残る |
| オブザーバビリティ | 内部が見える・動きが追える |
| ネイティブ | 中身で動く・組み込みの |
| プロキシ | 中継・外から見る |
| デリゲーション | 仕事任せ・依頼 |
| コンテキスト | 文脈・状況 |
| スキーマ | 設計図・構造 |
| I/O | 入力と出力 |
| レイテンシ | 遅延・時間差 |

## 投稿構造（必須）
① フック
② 何が起きたか（超簡潔）
③ 解説
④ 何がヤバいのか
⑤ 具体例
⑥ 未来の示唆
⑦ 行動誘導

## フックパターン（冒頭必須）
- 【海外で話題】
- 【速報】
- 【衝撃】
- 【注意喚起】
- 【保存版】
- 【実は】
- 【知らないと損】

## 読みやすさのルール
- 1行15〜25文字
- 改行多め
- 句読点少なめ
- スマホ可読性重視

悪い例：
```
OpenClawというAIエージェントは最近海外で話題になっていて様々な使い方が可能です
```

良い例：
```
OpenClawというAIエージェントが
海外でかなり話題になっています。
理由はシンプル。
「AIが自律で働く」から。
```

## 必須要素
1. **情報密度**
   - 何が起きたのか
   - 何ができるのか
   - 何が変わるのか

2. **箇条書き**
   途中で必ず箇条書きを入れる
   ```
   できること👇
   ・Web自動リサーチ
   ・競合分析
   ・ニュース収集
   ```

3. **ヤバさの言語化**
   「これ何がヤバいの？」を説明
   ```
   つまり何が起きるかというと…
   AIが
   「調べるツール」
   ↓
   「作るツール」
   ↓
   「働くツール」
   になり始めています。
   ```

4. **未来予測**
   必ず最後に未来視点
   ```
   これ
   本当に数年以内に
   「AIエージェントが当たり前」
   になる可能性あります。
   ```

5. **拡散トリガー**
   保存・引用・コメントを誘発
   ```
   正直これ
   かなり気になってるので
   実際に触ってみて
   レビューしようと思います。
   ```

6. **信頼感**
   企業名・技術名・数値を入れる
   ```
   ・GitHubで1300スター
   ・72%トークン削減
   ・Comet開発
   ```

## 投稿の長さ
- 通常：120〜300文字
- スレッド/まとめ：400〜800文字可

## よく使うパターン

### 型1：海外トレンド
```
【海外で話題】
〜が登場。
これ何かというと👇
〜説明
つまり…
〜解説
これ
かなりヤバいです。
```

### 型2：速報
```
【速報】
〜が公開。
結論👇
〜
できること👇
・
・
・
```

### 型3：保存版
```
【保存版】
〜の仕組みを
わかりやすく解説👇
```

## トーン
- フラット
- 断定しすぎない
- でもワクワク感

## 最後の一文
よく使うパターン：
- 「実際に触ってみてレビューします」
- 「使ってみて報告します」
- 「詳細解説します」

## 出力形式
1. 投稿案を5パターン作成
2. 各パターンは異なるフックで
3. HTMLページ生成
4. Vercelデプロイ
5. URLで共有
