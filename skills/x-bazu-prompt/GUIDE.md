# Xバズ投稿生成システム v2.0 実装ガイド

## システム概要

**入力:**
1. リライト対象のX投稿（テキスト or URL）
2. 自分のバズった投稿分析シート（X Premium or スプシデータ）

**出力:**
- 分析結果を学習した投稿案 5パターン
- HTMLページ（コピーボタン付き）
- Vercel URLで即座に共有可能

## バズ分析シートの作り方

### ステップ1: データ収集
X Premium or スプシで以下を記録：

```
【バズ分析シート】

投稿1:
- 日付: 2026-03-10
- フック: 【結論から言います】
- 文字数: 180
- いいね数: 5,347
- RT数: 892
- 構造: フック→箇条書き3点→つまり→未来予測→行動誘導
- 特徴: 無料配布系、学生向け、具体例あり
- 絵文字: 🔥使用

投稿2:
- 日付: 2026-03-08
- フック: 【速報】
- 文字数: 220
- いいね数: 3,871
- RT数: 645
- 構造: 速報→結論👇→箇条書き5点→つまり→行動誘導
- 特徴: 最新ニュース、先行性、権威性
- 絵文字: 🔥👇使用
```

### ステップ2: 傾向分析

```
【傾向まとめ】

■ フック効果ランキング
1位: 【結論から言います】- 平均4,200いいね
2位: 【速報】- 平均3,500いいね
3位: 正直- 平均2,800いいね

■ 最適文字数
- 平均: 185文字
- 範囲: 150-250文字

■ 必須要素
- 箇条書き（100%含む）
- 「つまり」での解説（90%含む）
- 未来予測（80%含む）

■ トーン傾向
- 親しみやすい（「正直」「思います」）
- 断定しすぎない（「可能性あります」）
- ワクワク感（「ヤバい」「劇的」）

■ 絵文字使用
- 🔥（最も効果的）
- 👇（箇条書き前に使用）
- 😳（驚きを演出）

■ 最終行パターン
- 「レビューします」（60%）
- 「検証してみます」（25%）
- 「解説します」（15%）
```

## システム使用フロー

### 通常時（分析シートなし）
```
1. いちさんがX投稿を送信
2. 標準パターン（AirCleベース）で5案作成
3. HTML生成 → URL共有
```

### 分析シートあり時（高精度モード）
```
1. いちさんがX投稿を送信
2. いちさんが分析シートを送信（または事前登録）
3. システムが分析シートを学習
4. 学習結果に基づき5案作成
5. HTML生成 → URL共有
```

## Tips

### バズる投稿を作るコツ

1. **分析シートは定期的に更新**
   - 月1回程度で傾向を確認
   - 新しいフックパターンを発見したら追加

2. **A/Bテストの記録**
   - 同じ内容でフックだけ変えた投稿
   - どれが効果的だったか記録

3. **季節・時期による変動**
   - 学生向け投稿は学期始めに強い
   - 時事ネタは速報型が強い

4. **競合分析も含める**
   - 他のAIアカウントのバズ投稿も参考に
   - 業界全体のトレンドを把握

## トラブルシューティング

### 投稿案がマンネリ化した時
→ 分析シートのフックパターンを新しいものに更新

### いいねが伸び悩んだ時
→ 文字数・絵emoji使用・「つまり」の有無を見直す

### 専門用語が多すぎる時
→ 噛み砕き表を参照し、難しい表現を置き換える
